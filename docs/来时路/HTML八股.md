## DOCTYPE文档类型
是html5中一种标准通用标记语言的文档类型声明，告诉浏览器应以html或xhtml的文档类型定义来解析文档，必须声明在html第一行。

浏览器渲染页面有两种方式，可通过`document.compatMode`获取：
- CSS1Compat：标准模式，默认模式，浏览器使用W3C标准解析页面，以支持的最高标准呈现页面
- BackCompat：混杂模式，页面以较宽松的向后兼容方式显示

## defer和async
- defer按顺序加载，按顺序执行，脚本执行发生在文档解析完成后、`DOMContentLoaded`时间触发执行前
- async不按顺序加载，哪个先加载完就先执行哪个

## 部分meta标签
### refresh
页面重定向与刷新：
```html
<meta http-equiv="refresh" content="5;url=https://www.baidu.com/">
```
页面每隔5秒自动跳转到百度首页。

### viewport
适配移动端，控制视口大小和比例：
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
`content`的参数有：
- width：宽度（数值 / device-width）
- height：高度（数值 / device-height）
- initial-scale：初始缩放比例
- maximum-scale：最大缩放比例
- minimum-scale：最小缩放比例
- user-scalable：是否允许用户缩放（yes / no）

### robots
搜索引擎索引方式：
```html
<meta name="robots" content="index, follow">
```
`content`的参数有：
- all：允许索引和跟踪（默认值）
- none：不允许索引和跟踪
- index：允许索引
- noindex：不允许索引
- follow：允许跟踪链接
- nofollow：不允许跟踪链接

## HTML5新增内容
### 语义化标签
- `<header>`：定义文档的页眉，通常包含网站标题、导航链接等
- `<nav>`：定义导航链接的部分，通常包含网站的主导航菜单
- `<main>`：定义文档的主要内容，页面中只能有一个`<main>`元素
- `<article>`：定义独立的内容块，适用于博客文章、新闻报道等
- `<section>`：定义文档中的一个区域，适用于章节、页眉、页脚等
- `<aside>`：定义与主内容相关的侧边内容，适用于侧边栏、广告等
- `<footer>`：定义文档的页脚，通常包含版权信息、联系信息等

### 媒体标签
#### `<audio>`
定义音频内容。
```html
<audio src='' controls autoplay loop></audio>
```
- `src`：音频文件的URL
- `controls`：显示浏览器默认的音频控件
- `autoplay`：页面加载后自动播放音频
- `loop`：音频播放结束后重新开始

#### `<video>`
定义视频内容。
```html
<video src='' poster controls autoplay loop></video>
```
- `src`：视频文件的URL
- `poster`：指定视频还没完全下载完毕或者用户没有点击播放按钮时显示的图片，默认为视频的第一帧
- `controls`：显示浏览器默认的视频控件
- `autoplay`：页面加载后自动播放视频
- `loop`：视频播放结束后重新开始

#### `<source>`
定义媒体资源的多个备选项。浏览器对视频格式支持程度不一致，使用`<source>`标签提供多个视频资源，浏览器会选择第一个支持的资源进行播放：
```html
<video>
    <source src='a.mp4' type='video/mp4'>
    <source src='a.flv' type='video/flv'>
</video>
```

### 表单
#### 表单类型
- email：输入必须是一个有效的电子邮件地址
- url：输入必须是一个有效的URL地址
- number：输入必须是一个数字，可以使用`min`、`max`和`step`属性限制输入范围和步长
- search：输入适合搜索的文本，通常会在输入框内显示一个清除按钮
- range：输入必须是一个范围内的数字，通常会显示一个滑动条，使用`min`、`max`和`step`属性限制输入范围和步长，`value`属性设置默认值
- color：输入必须是一个有效的颜色值，通常会显示一个颜色选择器
- time：时分秒输入，格式为hh:mm:ss
- date：年月日输入，格式为yyyy-mm-dd
- datetime：时间和日期，只有safari支持，格式为yyyy-mm-ddThh:mm:ssZ
- datetime-local：时间和日期，格式为yyyy-mm-ddThh:mm:ss，日期时间控件
- week：周控件
- month：月控件

#### 表单属性
- placeholder：输入提示文本，在输入框为空时显示
- autofocus：页面加载后自动聚焦到该输入框
- autocomplete：是否启用自动完成功能，值为on或off，有两个前提：
    - 输入框必须具有name属性
    - 表单必须提交过至少一次，浏览器才会记住输入的值并在下次输入时提供自动完成建议
- required：输入框必须填写内容才能提交表单
- pattern：输入必须匹配指定的正则表达式，格式为`pattern="正则表达式"`，例如`pattern="[A-Za-z]{3}"`表示输入必须是三个字母
- multiple：允许用户输入多个值，通常与email或file类型一起使用，表示用户可以输入多个电子邮件地址或选择多个文件，输入值之间用逗号分隔
- form：form表单的ID，允许将输入元素与一个不包含该元素的表单关联起来，例如`<input type="text" form="myForm">`表示该输入元素与ID为myForm的表单关联起来，即使该输入元素不在myForm表单内也可以提交到myForm表单中

#### 表单事件
- oninput：当用户输入内容时触发，适用于实时验证输入内容
- oninvalid：当输入内容不符合验证规则时触发，适用于显示错误提示信息

### 进度条、度量器
#### `<progress>`
定义一个进度条，表示任务的完成程度。用来表示任务的进度，IE和Safari不支持：
```html
<progress value="70" max="100"></progress>
```
- `value`：当前进度值，必须小于或等于max属性的值
- `max`：进度条的最大值，默认为1

#### `<meter>`
定义一个度量器，表示一个范围内的数值。用来表示一个数值在一个范围内的位置，IE和Safari不支持：
```html
<meter value="0.6" min="0" max="1"></meter>
```
- `value`：当前数值，必须在min和max属性指定的范围内
- `min`：数值的最小值，默认为0
- `max`：数值的最大值，默认为1
- `low`：数值的低端阈值，必须在min和max属性指定的范围内
- `high`：数值的高端阈值，必须在min和max属性指定的范围内
- `optimum`：数值的理想值，必须在min和max属性指定的范围内

设置规则：min < low < optimum < high < max

### Web存储
#### localStorage
- 存储数据没有过期时间，除非主动删除，否则数据一直存在
- 存储容量较大，通常为5MB或更多
- 数据以键值对的形式存储，键和值都是字符串类型
- 只能在同源的页面之间共享数据，不同域名、协议或端口之间的数据无法共享

#### sessionStorage
- 存储数据在页面会话期间有效，页面会话结束时数据被删除，例如关闭浏览器标签页或窗口
- 存储容量较小，通常为5MB或更少
- 数据以键值对的形式存储，键和值都是字符串类型
- 只能在同源的页面之间共享数据，不同域名、协议或端口之间的数据无法共享

## img标签的srcset属性
响应式页面经常需要根据屏幕密度设置不同的图片资源，`srcset`属性允许我们为不同的屏幕密度提供不同的图片资源，浏览器会根据设备的屏幕密度选择合适的图片进行加载，从而提高页面的性能和用户体验。
```html
<img src="image-128.png" srcset="image-256.png 2x" alt="示例图片">  
```
以上标签在屏幕密度为1x时加载`image-128.png`，在屏幕密度为2x时加载`image-256.png`。

当有多种屏幕密度时可以写成：
```html
<img 
    src="image-128.png" 
    srcset="image-128.png 128w, image-256.png 256w, image-384.png 384w" 
    sizes="(max-width: 600px) 128px, (max-width: 1200px) 256px, 384px"
    alt="示例图片"
/>
```
srcset用来设置图片的资源和对应的图片质量，sizes用来设置图片的尺寸临界点，单位`w`可以理解为图片质量，当可视区域小于这个质量的值就可以使用。浏览器会自动选择最小可用的图片资源进行加载。

sizes语法是
```html
sizes="(media condition) size, (media condition) size, ... , size"
```
上述的img标签在可视区域宽度小于600px时加载`image-128.png`，在可视区域宽度小于1200px时加载`image-256.png`，否则加载`image-384.png`。

## 行内元素
`a`, `b`, `br`, `code`, `em`, `i`, `img`, `input`, `label`, `select`, `span`, `strong`, `sub`, `sup`等。行内元素不会在前后产生换行，宽度由内容决定，可以设置`padding`和`margin`，但不能设置`width`和`height`。

`div`, `h1`-`h6`, `p`, `ul`, `ol`, `li`, `table`, `header`, `nav`, `main`, `article`, `section`, `aside`, `footer`等。块级元素会在前后产生换行，宽度默认为父元素的100%，可以设置`width`和`height`。

## 空元素
### 常见的
`br`, `hr`, `img`, `input`, `meta`, `link`等。空元素没有内容，不能有子元素，不能设置`width`和`height`，但可以设置`padding`和`margin`。
### 不常见的    
`area`, `base`, `col`, `embed`, `param`, `source`, `track`, `wbr`等。空元素没有内容，不能有子元素，不能设置`width`和`height`，但可以设置`padding`和`margin`。

## Web Worker的理解
在HTML页面内，执行脚本时页面的状态是不可响应的，知道脚本执行完成，页面才变为可响应。Web Worker是运行在后台的js线程，独立于其他脚本，不会影响页面性能。并通过postMessage方法与主线程进行通信，适用于需要执行大量计算或处理大量数据的任务，例如图像处理、数据分析等。这样就不会阻塞主线程，保持页面的响应性和流畅性。

创建方法：
1. 检测浏览器是否支持Web Worker
2. 创建Web Worker文件：js、回调函数等
3. 创建Web Worker实例：`const worker = new Worker('worker.js');`

## HTML5离线储存
离线储存指的是在用户没有与网络连接时，可以正常访问站点或应用，在用户联网时更新用户机器上的缓存文件。

### 原理
HTML5的离线存储是基于一个新建的`.appcache`文件实现的，`.appcache`文件是一个文本文件，包含了需要缓存的资源列表和一些配置选项。当用户访问一个使用了离线储存的站点时，浏览器会下载`.appcache`文件，并根据其中的资源列表将指定的资源缓存到本地。当用户再次访问该站点时，如果没有网络连接，浏览器会从本地缓存中加载资源，从而实现离线访问。

### 使用方法
创建一个和HTML文件同名的`.appcache`文件，例如`index.html`对应`index.appcache`，并在HTML文件的第一行添加以下代码：
```html
<html lang="en" manifest="index.appcache">
```

在`.appcache`文件中，按照以下格式编写需要缓存的资源列表：
```CACHE MANIFEST
# 版本号
# v0.11
# 资源列表
CACHE:
index.html
style.css
script.js
image.png
# 网络资源列表
NETWORK:
*
FALLBACK:
```
- `CACHE`：需要缓存的资源列表，可以是相对路径或绝对路径，由于包含manifest属性的HTML文件会被自动缓存，所以可以不写在CACHE列表中
- `NETWORK`：需要在线访问的资源列表，使用`*`表示所有资源都需要在线访问，他们不会被离线存储，在离线状态下访问这些资源会失败。如果在CACHE列表中已经缓存了某个资源，那么即使在NETWORK列表中也不会被在线访问，而是直接从离线存储中加载，也就是CACHE列表优先于NETWORK列表
- `FALLBACK`：当用户访问一个离线资源时失败，可以指定一个替代资源来提供给用户。

在离线状态下，操作`window.applicationCache`对象可以获取离线存储的状态，进行离线缓存的操作。

### 如何更新缓存
- 更新manifest文件：当需要更新缓存时，可以修改`.appcache`文件中的内容，例如添加一个新的资源或者修改版本号。浏览器会检测到`.appcache`文件的变化，并重新下载和缓存新的资源。
- 通过javascript更新缓存：可以通过调用`window.applicationCache.update()`方法来手动触发缓存更新。当调用该方法时，浏览器会检查`.appcache`文件是否有变化，如果有变化则重新下载和缓存新的资源。
- 清除缓存：可以通过调用`window.applicationCache.clear()`方法来清除浏览器中的离线缓存。当调用该方法时，浏览器会删除所有与当前页面相关的离线缓存资源。

### 注意事项
- 浏览器对缓存数据的容量不一样，通常为5MB或更多，超过容量限制时会抛出异常。
- 如果manifest文件内有资源无法访问，整个离线存储将无法使用，因此需要确保manifest文件中的资源都可以正常访问。
- 引用manifest文件的HTML文件必须与manifest文件在同一域名、协议和端口下，否则浏览器会拒绝加载manifest文件，导致离线存储无法使用。
- FALLBACK中的资源必须和manifest文件在同一域名、协议和端口下，否则浏览器会拒绝加载替代资源，导致离线存储无法使用。
- 当一个资源被缓存后，浏览器直接请求这个绝对路径也可以访问缓存中的资源。
- 站点中的其他页面即使没有设置manifest属性，请求的资源如果在缓存中也会直接从缓存中加载。
- 当manifest文件改变时，资源请求本身也会触发更新，

### 浏览器如何管理和加载离线存储资源
#### 在线情况下
浏览器发现HTML头部有manifest属性时，会下载manifest文件，如果第一次访问页面，浏览器就会根据manifest文件中的资源列表将指定的资源缓存到本地。如果已经访问过页面，浏览器会检查manifest文件是否有变化，如果有变化则重新下载和缓存新的资源。

#### 离线情况下
浏览器直接使用离线存储的资源。

## title与h1的区别
title属性没有明确意义只表示是标题，h1标签表示是一级标题，具有语义化意义，搜索引擎会根据h1标签的内容来判断页面的主题和内容，因此在SEO优化中，合理使用h1标签可以提高页面的搜索引擎排名和可见性。

## b与strong的区别
strong标签有语义化意义，表示强调的内容，搜索引擎会根据strong标签的内容来判断页面的重要性和相关性，因此在SEO优化中，合理使用strong标签可以提高页面的搜索引擎排名和可见性，而b标签没有语义化意义，只是简单地将文本加粗，不会对SEO产生影响。

## i与em的区别
i表示斜体，em表示强调，em标签有语义化意义，表示强调的内容，搜索引擎会根据em标签的内容来判断页面的重要性和相关性，因此在SEO优化中，合理使用em标签可以提高页面的搜索引擎排名和可见性，而i标签没有语义化意义，只是简单地将文本斜体，不会对SEO产生影响。

## iframe的优缺点
iframe会创建包含另外一个文档的内联框架，可以在一个页面中嵌入另一个页面。
### 优点
- 用来加载速度较慢的内容（如广告），不会阻塞主页面的加载
- 可以跨域加载内容，适用于需要嵌入第三方内容的场景，例如社交媒体插件、地图等
- 可以实现脚本并行下载
### 缺点
- iframe会阻塞主页面的onload事件，导致页面加载时间变长
- 无法被一些搜索引擎索引，可能会对SEO产生负面影响
- 会产生很多页面，增加页面的复杂性和维护成本

## label标签
label标签用来定义表单控件的关系：当用户选择label标签时，浏览器会自动将焦点移到与该label标签关联的表单控件上，从而提高用户体验和可访问性。label标签有两种关联方式：
- 包裹式：将表单控件包裹在label标签内，例如
```html
<label>用户名：<input type="text" name="username"></label>
```
- for属性：使用for属性将label标签与表单控件关联起来，例如

## Canvas和SVG的区别
注：矢量图，也称为面向对象的图像或绘图图像，在数学上定义为一系列由线连接的点。矢量文件中的图形元素称为对象。每个图像都是一个自成一体的实体，它具有自己的属性，例如位置、大小、颜色等。矢量图像可以无限缩放而不会失真，因为它们是基于数学公式定义的，而不是基于像素的位图图像。

### Canvas
Canvas画布，通过JavaScript绘制图形，逐像素进行渲染，其位置改变则发生重绘。

特点是：
- 依赖分辨率
- 不支持事件处理器
- 弱的文本渲染能力
- 能过以.png或.jpg格式导出图像
- 适合图像密集型的应用，例如游戏、数据可视化等。

### SVG
SVG（Scalable Vector Graphics）是一种基于XML的矢量图形格式，即可伸缩矢量图形，图像在放大或缩小时不会失真，适用于图标、图表等需要保持清晰度的图形。SVG基于XML就意味着SVG DOM中的每个元素都是一个对象，可以通过JavaScript来操作这些对象，从而实现动态效果和交互功能。如果SVG对象的属性发生变化，浏览器会自动重新渲染该对象，从而实现动态效果和交互功能。

特点是：
- 不依赖分辨率
- 支持事件处理器
- 适合带有大型渲染区域的应用，例如地图、图表等。
- 复杂度高会减慢渲染速度，适合简单图形的应用，例如图标、标志等。
- 不适合图像密集型的应用，例如游戏、数据可视化等。

## head标签
head标签包含了页面的元数据，例如标题、字符编码、样式表、脚本等。head标签中的内容不会直接显示在页面上，但会影响页面的行为和外观。head标签的作用是提供页面的元信息，帮助浏览器正确地解析和渲染页面，同时也有助于搜索引擎优化和用户体验的提升。head标签中的内容可以包括以下几个方面：
- 标题：使用title标签定义页面的标题，显示在浏览器的标签页上，也是搜索引擎结果中的标题。
- 字符编码：使用meta标签定义页面的字符编码，确保页面能够正确显示各种语言的字符。
- 样式表：使用link标签引入外部样式表，或者使用style标签定义内部样式表，控制页面的外观和布局。
- 脚本：使用script标签引入外部脚本，或者使用script标签定义内部脚本，控制页面的行为和交互功能。
- 其他元信息：使用meta标签定义页面的描述、关键词、作者等元信息，帮助搜索引擎优化和用户体验的提升。

可用于head部分的标签有：`<base>`, `<link>`, `<meta>`, `<script>`, `<style>`, `<title>`，head标签必不可少的是title标签和meta标签，其他标签根据需要选择使用。

## Doctype和`<!Doctype html>`的作用
- Doctype：文档声明是为了告诉浏览器当前HTML文档使用什么版本的HTML来写的，这样浏览器才能按照声明的版本来正确的解析。
- `<!Doctype html>`：`<!doctype html>`的作用就是让浏览器进入标准模式，使用最新的 HTML5 标准来解析渲染页面；如果不写，浏览器就会进入混杂模式，需要避免此类情况发生。

### 严格模式与混杂模式
- 严格模式：又称为标准模式，指浏览器按照W3C标准解析代码；
- 混杂模式：又称怪异模式、兼容模式，是指浏览器用自己的方式解析代码。混杂模式通常模拟老式浏览器的行为，以防止老站点无法工作；

区分：网页中的DTD（Document Type Definition），直接影响到使用的是严格模式还是浏览模式，可以说DTD的使用与这两种方式的区别息息相关。
- 如果文档包含严格的DOCTYPE ，那么它一般以严格模式呈现（严格 DTD ——严格模式）；
- 包含过渡 DTD 和 URI 的 DOCTYPE ，也以严格模式呈现，但有过渡 DTD 而没有 URI （统一资源标识符，就是声明最后的地址）会导致页面以混杂模式呈现（有 URI 的过渡 DTD ——严格模式；没有 URI 的过渡 DTD ——混杂模式）；
- DOCTYPE 不存在或形式不正确会导致文档以混杂模式呈现（DTD不存在或者格式不正确——混杂模式）；
- HTML5 没有 DTD ，因此也就没有严格模式与混杂模式的区别，HTML5 有相对宽松的 法，实现时，已经尽可能大的实现了向后兼容(HTML5 没有严格和混杂之分)。

严格模式让各个浏览器统一执行一套规范兼容模式保证了旧网站的正常运行。

## 浏览器乱码
### 产生乱码的原因
- 网页源代码是gbk的编码，而内容中的中文字是utf-8编码的，这样浏览器打开即会出现html乱码，反之也会出现乱码；
- html网页编码是gbk，而程序从数据库中调出呈现是utf-8编码的内容也会造成编码乱码；
- 浏览器不能自动检测网页编码，造成网页乱码。

### 解决方法
- 使用软件编辑HTML网页内容；
- 如果网页设置编码是gbk，而数据库储存数据编码格式是UTF-8，此时需要程序查询数据库数据显示数据前进程序转码；
- 如果浏览器浏览时候出现网页乱码，在浏览器中找到转换编码的菜单进行转换。

## 渐进增强和优雅降级
- 渐进增强（progressive enhancement）：主要是针对低版本的浏览器进行页面重构，保证基本的功能情况下，再针对高级浏览器进行效果、交互等方面的改进和追加功能，以达到更好的用户体验。
- 优雅降级（graceful degradation）：一开始就构建完整的功能，然后再针对低版本的浏览器进行兼容。

两者区别：
- 优雅降级是从复杂的现状开始的，并试图减少用户体验的供给；而渐进增强是从一个非常基础的，能够起作用的版本开始的，并在此基础上不断扩充，以适应未来环境的需要；
- 降级（功能衰竭）意味着往回看，而渐进增强则意味着往前看，同时保证其根基处于安全地带。

“优雅降级”观点认为应该针对那些最高级、最完善的浏览器来设计网站。而将那些被认为“过时”或有功能缺失的浏览器下的测试工作安排在开发周期的最后阶段，并把测试对象限定为主流浏览器（如 IE、Mozilla 等）的前一个版本。 在这种设计范例下，旧版的浏览器被认为仅能提供“简陋却无妨 (poor, but passable)” 的浏览体验。可以做一些小的调整来适应某个特定的浏览器。但由于它们并非我们所关注的焦点，因此除了修复较大的错误之外，其它的差异将被直接忽略。

“渐进增强”观点则认为应关注于内容本身。内容是建立网站的诱因，有的网站展示它，有的则收集它，有的寻求，有的操作，还有的网站甚至会包含以上的种种，但相同点是它们全都涉及到内容。这使得“渐进增强”成为一种更为合理的设计范例。这也是它立即被 Yahoo 所采纳并用以构建其“分级式浏览器支持 (Graded Browser Support)”策略的原因所在。

## drag API
- dragstart：事件主体是被拖放元素，在开始拖放被拖放元素时触发。
- drag：事件主体是被拖放元素，在正在拖放被拖放元素时触发。
- dragenter：事件主体是目标元素，在被拖放元素进入某元素时触发。
- dragover：事件主体是目标元素，在被拖放在某元素内移动时触发。
- dragleave：事件主体是目标元素，在被拖放元素移出目标元素是触发。
- drop：事件主体是目标元素，在目标元素完全接受被拖放元素时触发。
- dragend：事件主体是被拖放元素，在整个拖放操作结束时触发。
