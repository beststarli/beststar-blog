# 防抖与节流

防抖和节流是JavaScript中处理高频事件（如用户交互、滚动、调整大小）的常用优化技巧，用于避免函数过度执行导致性能问题。

## 防抖Debounce
### 概念
将高频事件合并为一次执行，等待事件停止触发一段时间（延迟时间）后才执行函数。适合“最后一次”才处理的场景。

防抖的原理是在事件被触发后，在指定的时间内如果事件没有再次被触发，则执行一次函数；如果在这段时间内事件再次被触发，则重新计时。简单来说，就是将多次触发的事件合并为一次执行，常用于需要等待用户操作完全停止后再进行处理的场景，如输入框搜索、窗口大小调整等。

## 节流Throttle
### 概念
限制函数执行频率，在指定时间间隔内只执行一次，适合“均匀间隔”执行的场景。

节流的原理是在规定的时间间隔内，无论事件触发了多少次，都只执行一次函数。也就是说，它会确保函数以一定的频率执行，而不是无限制地响应事件触发。节流常用于需要频繁但有规律地执行某些操作的场景，如滚动监听、鼠标移动等。
