# [350.两个数组的交集II](https://leetcode.cn/problems/intersection-of-two-arrays-ii/description/)   
给你两个整数数组 `nums1` 和 `nums2` ，请你以数组形式返回两数组的交集。返回结果中每个元素出现的次数，应与元素在两个数组中都出现的次数一致（如果出现次数不一致，则考虑取较小值）。可以不考虑输出结果的顺序。

示例1：
- **输入**：nums1 = [1,2,2,1], nums2 = [2,2]
- **输出**：[2,2]

示例2：
- **输入**：nums1 = [4,9,5], nums2 = [9,4,9,8,4]
- **输出**：[4,9]

**进阶**：
- 如果给定的数组已经排好序呢？你将如何优化你的算法？
- 如果 `nums1` 的大小比 `nums2` 小，哪种方法更优？
- 如果 `nums2` 的元素存储在磁盘上，内存是有限的，并且你不能一次加载所有的元素到内存中，你该怎么办？

## 解题思路
### 1.哈希map频率表
与[349.两个数组的交集](./349.两个数组的交集.md)的区别在于，349题中只需要返回交集中的唯一元素，而350题中需要返回交集中的所有元素，并且每个元素出现的次数应与元素在两个数组中都出现的次数一致（如果出现次数不一致，则考虑取较小值）。那么就不能单纯的使用哈希set来记录一次了，应改为用哈希map来记录每个元素出现的次数，然后遍历另一个数组，判断该元素是否在map中存在，如果存在且频率大于0，则将其加入结果数组，并将map中该元素的次数减1。

- 时间复杂度：O(n + m)，其中 n 和 m 分别是数组 nums1 和 nums2 的长度。我们需要遍历两个数组来构建频率表和生成结果。
- 空间复杂度：O(min(n, m))，我们使用了一个哈希map来存储较小数组的元素及其频率。

```js
function intersect(nums1: number[], nums2: number[]): number[] {
    const count = new Map()
    nums1.forEach(num => count.set(num, (count.get(num) ?? 0) + 1))
    const ans = []
    nums2.forEach(num => {
        const times = count.get(num) ?? 0
        if (times > 0) {
            ans.push(num)
            count.set(num, times - 1)
        }
    })
    return ans
};
```

### 2.双指针
既然题目不要求返回的结果数组有顺序，那么双指针的写法则需要对两个数组先进行升序排序，分别初始化两个指针i和j，分别指向两个数组的开头，然后进行如下操作：
- 如果nums1[i] < nums2[j]，则代表nums2中以没有当前nums1中所指的数字，i++
- 如果nums1[i] > nums2[j]，则代表nums1中以没有当前nums2中所指的数字，j++
- 如果nums1[i] == nums2[j]，则代表找到了一个交集元素，将其加入结果数组，并且i++，j++

- 时间复杂度：O(nlogn + mlogm)，其中 n 和 m 分别是数组 nums1 和 nums2 的长度。我们需要对两个数组进行排序，然后使用双指针遍历两个数组。
- 空间复杂度：O(1)，不需要额外的空间来存储数据结构，除了返回结果数组。

```js
function intersect(nums1: number[], nums2: number[]): number[] {
    nums1.sort((a, b) => a - b)
    nums2.sort((a, b) => a - b)
    const ans = []
    let i = 0
    let j = 0
    while (i < nums1.length && j < nums2.length) {
        const num1 = nums1[i]
        const num2 = nums2[j]
        if (num1 < num2) {
            i++
        } else if (num1 > num2) {
            j++
        } else {
            ans.push(num1)
            i++
            j++
        }
    }
    return ans
};
```

## 总结
### 进阶一
改用双指针解决，进行空间优化，空间复杂度为O(1)。
### 进阶二
如果依然是使用哈希表，那么对于较小的数组，使用哈希表来存储其元素及频率，而对于较大的数组，直接遍历并查找即可。
### 进阶三
用一个小型缓冲区（buffer）一边读数据一边遍历数据。这等价于问 nums<sub>2</sub>是一个流（Stream）的情况要怎么做。由于我们写的是一次遍历的代码，所以已经符合这个要求。
