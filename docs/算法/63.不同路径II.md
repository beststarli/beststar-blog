# [63.不同路径II](https://leetcode.cn/problems/unique-paths-ii/description/)   
给定一个 `m x n` 的整数数组 `grid`。一个机器人初始位于 **左上角**（即 `grid[0][0]`）。机器人尝试移动到 **右下角**（即 `grid[m - 1][n - 1]`）。机器人每次只能向下或者向右移动一步。

网格中的障碍物和空位置分别用 `1` 和 `0` 来表示。机器人的移动路径中不能包含 任何 有障碍物的方格。

返回机器人能够到达右下角的不同路径数量。

测试用例保证答案小于等于 `2 * 109`。

示例1：
![63-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/63-1.jpg)
- **输入**：obstacleGrid = [[0,0,0],[0,1,0],[0,0,0]]
- **输出**：2
- **解释**：
```txt
3x3 网格的正中间有一个障碍物。
从左上角到右下角一共有 2 条不同的路径：
1. 向右 -> 向右 -> 向下 -> 向下
2. 向下 -> 向下 -> 向右 -> 向右
```

- **输出**：1

## 解题思路
### 1.递归加记忆化
与[62.不同路径](/docs/算法/62.不同路径.md)相比，只是多了障碍物的条件判断，只需要当遇到障碍物时，将到达障碍物所在格子路径的数量设置为0即可，也就是`memo[i][j] = 0`，其他的逻辑不变。

- 时间复杂度：O(mn)，其中 m 和 n 分别是网格的行数和列数。我们需要计算每个格子的路径数量，每个格子只计算一次。
- 空间复杂度：O(mn)，其中 m 和 n 分别是网格的行数和列数。我们使用了一个大小为 m x n 的数组来存储每个格子的路径数量。

```ts
function uniquePathsWithObstacles(obstacleGrid: number[][]): number {
    const m = obstacleGrid.length
    const n = obstacleGrid[0].length
    const memo = Array.from({ length: m }, () => Array(n).fill(-1))

    function dfs(i, j) {
        if (i < 0 || j < 0) {
            return 0
        }
        if (obstacleGrid[i][j] === 1) {
           return memo[i][j] = 0
        }
        if (i === 0 && j === 0) {
            return 1
        }
        if (memo[i][j] !== -1) {
            return memo[i][j]
        }
        memo[i][j] = dfs(i - 1, j) + dfs(i, j - 1)
        return memo[i][j]
    }
    return dfs(m - 1, n - 1)
};
```

### 2.递推
一比一翻译递归为递推，只是针对memo数组需要在左侧和上方额外添加一行和一列，初始化为0，表示到达障碍物的路径数量为0。然后从索引1开始计算每个格子的路径数量，如果遇到障碍物，则将该格子的路径数量设置为0。先判断是否是障碍，是的话赋值后跳过本轮；再判断是否是起点，是的话赋值后跳过本轮；否则就计算当前格子的路径数量。

- 时间复杂度：O(mn)，其中 m 和 n 分别是网格的行数和列数。我们需要计算每个格子的路径数量，每个格子只计算一次。
- 空间复杂度：O(mn)，其中 m 和 n 分别是网格的行数和列数。我们使用了一个大小为 m x n 的数组来存储每个格子的路径数量。

```ts
function uniquePathsWithObstacles(obstacleGrid: number[][]): number {
    const m = obstacleGrid.length
    const n = obstacleGrid[0].length
    const memo = Array.from({ length: m + 1 }, () => Array(n + 1).fill(0))
    

    for (let i = 1; i <= m; i++) {
        for (let j = 1; j <= n; j++) {
            if (obstacleGrid[i - 1][j - 1] === 1) {
                memo[i][j] = 0
                continue
            }
            if (i === 1 && j === 1) {
                memo[i][j] = 1
                continue
            }
            memo[i][j] = memo[i - 1][j] + memo[i][j - 1]
        }
    }
    return memo[m][n]
};
```

### 3.空间优化
还是格子路径累加的思路，用一个一维数组来存储每一列的路径数量，初始化为0，表示到达障碍物的路径数量为0。然后从索引1开始计算每个格子的路径数量，如果遇到障碍物，则将该格子的路径数量设置为0。先判断是否是障碍，是的话赋值后跳过本轮；否则就计算当前格子的路径数量。注意初始化第一个格子为1，表示只有一个格子时也看作是一条路径。

- 时间复杂度：O(mn)，其中 m 和 n 分别是网格的行数和列数。我们需要计算每个格子的路径数量，每个格子只计算一次。
- 空间复杂度：O(n)，其中 n 是网格的列数。我们使用了一个大小为 n 的数组来存储每一列的路径数量。

```ts
function uniquePathsWithObstacles(obstacleGrid: number[][]): number {
    const m = obstacleGrid.length
    const n = obstacleGrid[0].length
    const memo = Array(n + 1).fill(0)
    memo[1] = 1
    for (let i = 0; i < m; i++) {
        for (let j = 1; j <= n; j++) {
            if (obstacleGrid[i][j - 1] === 1) {
                memo[j] = 0
            } else {
                memo[j] += memo[j - 1]
            }
        }
    }
    return memo[n]
};
```
