# [925.长按键入](https://leetcode.cn/problems/long-pressed-name/description/)
你的朋友正在使用键盘输入他的名字 `name`。偶尔，在键入字符 `c` 时，按键可能会被长按，而字符可能被输入 1 次或多次。

你将会检查键盘输入的字符 `typed`。如果它对应的可能是你的朋友的名字（其中一些字符可能被长按），那么就返回 `true`。

示例1：
- **输入**：name = "alex", typed = "aaleex"
- **输出**：true
- **解释**：'alex' 中的 'a' 和 'e' 被长按。

示例2：
- **输入**：name = "saeed", typed = "ssaaedd"
- **输出**：false
- **解释**：'e' 一定需要被键入两次，但在 'typed' 中不是这样。

## 解题思路
### 双指针
本质上这是个隐式双指针，每个字符串有个指针指向当前字符，以typed作为基准，如果typed的当前字符和name的当前字符相同，则两个指针都向后移动；如果typed的当前字符和name的前一个字符相同，则typed指针向后移动；否则返回false。最终判断name指针是否到达末尾，如果到达则说明typed是name的长按键入。

- 时间复杂度：O(n)，其中 n 是字符串数组的长度。
- 空间复杂度：O(1)，只使用了常数空间。

```ts
function isLongPressedName(name: string, typed: string): boolean {
    let pointer = 0
    for(const char of typed) {
        if (char === name[pointer]) {
            pointer++
        } else if (pointer > 0 && char === name[pointer - 1]) {
            continue
        } else {
            return false
        }
    }
    return pointer === name.length
};
```
