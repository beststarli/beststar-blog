# [203.移除链表元素](https://leetcode.cn/problems/remove-linked-list-elements/description/)
给你一个链表的头节点 `head` 和一个整数 `val` ，请你删除链表中所有满足 `Node.val == val` 的节点，并返回 **新的头节点**。

示例1：
![203](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/203.jpg)
- **输入**：head = [1,2,6,3,4,5,6], val = 6
- **输出**：[1,2,3,4,5]

示例2：
- **输入**：head = [], val = 1
- **输出**：[]

示例3：
- **输入**：head = [7,7,7,7], val = 7
- **输出**：[]

## 解题思路
### 1.守卫节点
既然是返回头节点，就需要建立一个守卫节点dummy，它的next指向head，然后用一个指针p0指向head，遍历链表，如果p0的下一个节点的值等于val，就将p0的next指向p0的下一个节点的下一个节点，否则就将p0指向下一个节点。最后返回dummy.next即可。

- 时间复杂度：O(n)，其中 n 是链表的长度。我们需要遍历链表一次来删除满足条件的节点。
- 空间复杂度：O(1)。我们只使用了常数级别的额外空间来存储指针。

```ts
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     val: number
 *     next: ListNode | null
 *     constructor(val?: number, next?: ListNode | null) {
 *         this.val = (val===undefined ? 0 : val)
 *         this.next = (next===undefined ? null : next)
 *     }
 * }
 */

function removeElements(head: ListNode | null, val: number): ListNode | null {
    const dummy = new ListNode(0, head)
    let p0 = dummy
    while (p0.next !== null) {
        if (p0.next.val === val) {
            p0.next = p0.next.next
        } else {
            p0 = p0.next
        }
    }
    return dummy.next
};
```

## 总结
为什么在p0.next为null时就停止遍历呢？因为如果p0.next为null，说明p0已经是链表的最后一个节点了，那么就没有必要再继续遍历了。我们这里要处理的也是p0.next的值是否等于val，所以当p0.next为null时，就说明已经没有下一个节点了，也就没有必要再继续遍历了。
