# [62.不同路径](https://leetcode.cn/problems/unique-paths/)  
一个机器人位于一个`m x n`网格的左上角 （起始点在下图中标记为 “Start” ）。

机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。

问总共有多少条不同的路径？

示例1：
![62-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/62-1.png)
- **输入**：m = 3, n = 7
- **输出**：28

示例2：
- **输入**：m = 3, n = 2
- **输出**：3
- **解释**：
```txt
从左上角开始，总共有 3 条路径可以到达右下角。
1. 向右 -> 向下 -> 向下
2. 向下 -> 向下 -> 向右
3. 向下 -> 向右 -> 向下
```

示例3：
- **输入**：m = 7, n = 3
- **输出**：28

示例4：
- **输入**：m = 3, n = 3
- **输出**：6

## 解题思路
### 1.组合
机器人需要向下移动`m-1`步，向右移动`n-1`步，总共需要移动`m+n-2`步。在这`m+n-2`步中选择`m-1`步向下移动，剩余的就是向右移动。因此不同路径的数量等于从`m+n-2`步中选择`m-1`步的组合数，即 `C(m+n-2, m-1)`。如果令`width = m - 1`，`height = n - 1`，则不同路径的数量等于 `C(width + height, width)`。问题在于如果直接用辅助函数来实现阶乘，可能会导致整数溢出。我们可以在计算组合数的过程中进行约分，避免中间结果过大。

对于阶乘公式：
$$
C(a, b) = \frac{a!}{b! \times (a - b)!} = \frac{(a - b + 1) \times (a - b + 2) \times \cdots \times a}{1 \times 2 \times \cdots \times b}
$$

令 $a = width + height$，$b = width$（假设 $width \leq height$），则：

$$
C(width + height, width) = \frac{(height + 1) \times (height + 2) \times \cdots \times (height + width)}{1 \times 2 \times \cdots \times width}
$$

于是我们将上式中的分子和分母进行约分，就可以通过遍历来计算组合数，避免了中间结果过大的问题。遍历的次数就是 `width`，每次迭代中分子乘以 `height + i`，分母乘以 `i`，最终得到的结果就是不同路径的数量。

时间复杂度：$O(m)$，其中 m 是网格的行数。

```ts
function uniquePaths(m: number, n: number): number {
    if (m > n) {
        [m, n] = [n, m];
    }

    const width = m - 1
    const height = n - 1
    let ans = 1

    for (let i = 1; i <= width; i++) {
        ans = ans * (height + i) / i;
    }

    return ans
};
```

### 2.递归加记忆化
定义`dfs(i, j)`，表示从位置`(i, j)`出发到达右下角的不同路径数量。如果此时移动到`[i - 1, j]`或者`[i, j - 1]`，那么由加法原理，`dfs(i, j)`可以由`dfs(i - 1, j) + dfs(i, j - 1)`转移而来。对原数组中的每个位置都调用一次`dfs(i, j)`，最终的答案是`dfs(m - 1, n - 1)`。

于是就有了递归入口dfs(m - 1, n - 1)，递归出口dfs(0, 0) = 1，递归关系dfs(i, j) = dfs(i - 1, j) + dfs(i, j - 1)。递归边界是当`i < 0`或者`j < 0`时，返回0。

时间复杂度：$O(mn)$，其中 m 和 n 分别是网格的行数和列数。

```ts
function uniquePaths(m: number, n: number): number {
    const memo = Array.from({ length: m }, () => Array(n).fill(0))
    return dfs(m - 1, n - 1, memo)
};

function dfs(i, j, memo) {
    if (i < 0 || j < 0) {
        return 0
    }
    if (i === 0 && j === 0) {
        return 1
    }
    if (memo[i][j] !== 0) {
        return memo[i][j]
    }

    return memo[i][j] = dfs(i - 1, j, memo) + dfs(i, j - 1, memo) 
}
```

### 3.递推
还是一比一翻译，保留递归中「归」的部分。用`memo[i + 1][j + 1]`表示从位置`(i, j)`出发到达右下角的不同路径数量。递推的入口是`memo[1][1] = 1`，递推关系是`memo[i + 1][j + 1] = memo[i][j + 1] + memo[i + 1][j]`，其中`i = 0,1,...,m - 1`且`j = 0,1,...,n - 1`。最终的答案是`memo[m][n]`。

时间复杂度：$O(mn)$，其中 m 和 n 分别是网格的行数和列数。

空间复杂度：$O(mn)$，其中 m 和 n 分别是网格的行数和列数。

```ts
function uniquePaths(m: number, n: number): number {
    const memo = Array.from({ length: m + 1 }, () => Array(n + 1).fill(0))
    memo[0][0] = 1
    
    for (let i = 0; i < m; i++) {
        for (let j = 0; j < n; j++) {
            if (i === 0 && j === 0) {
                memo[1][1] = 1
            } else{
                memo[i + 1][j + 1] = memo[i][j + 1] + memo[i + 1][j]
            }
        }
    }

    return memo[m][n]
};
```

### 4.空间优化
对于[解法3](/docs/算法/不同路径#3递推)，我们可以发现`memo[i + 1][j + 1]`的值只与`memo[i][j + 1]`和`memo[i + 1][j]`有关，因此我们可以使用一个一维数组来进行空间优化。用`memo[j + 1]`表示从位置`(i, j)`出发到达右下角的不同路径数量。递推的入口是`memo[1] = 1`，递推关系是`memo[j + 1] += memo[j]`，其中`i = 0,1,...,m - 1`且`j = 0,1,...,n - 1`。最终的答案是`memo[n]`。


时间复杂度：$O(mn)$，其中 m 和 n 分别是网格的行数和列数。

空间复杂度：$O(n)$，其中 n 是网格的列数。

```ts
function uniquePaths(m: number, n: number): number {
    const memo = Array(n + 1).fill(0)
    memo[1] = 1

    for (let i = 0; i < m; i++) {
        for (let j = 0; j < n; j++) {
            memo[j + 1] += memo[j]
        }
    }

    return memo[n]
};
```

## 二刷
### 1.递归加记忆化优化
注意memo数组的初始化`memo[0][0] = 1`，也就是只有一个格子时也看作是一条路径。

- 时间复杂度：$O(mn)$，其中 m 和 n 分别是网格的行数和列数。
- 空间复杂度：$O(mn)$，其中 m 和 n 分别是网格的行数和列数。

```ts
function uniquePaths(m: number, n: number): number {
    const memo = Array.from({ length: m }, () => Array(n).fill(0))
    memo[0][0] = 1
    
    function dfs(i, j) {
        if (i < 0 || j < 0) {
            return 0
        }
        if (i === 0 && j === 0) {
            return 1
        }
        if (memo[i][j] !== 0) {
            return memo[i][j]
        }
        memo[i][j] = dfs(i - 1, j) + dfs(i, j - 1)
        return memo[i][j]
    }
    dfs(m - 1, n - 1)
    return memo[m - 1][n - 1]
};
```

### 2.递推
- 时间复杂度：$O(mn)$，其中 m 和 n 分别是网格的行数和列数。
- 空间复杂度：$O(mn)$，其中 m 和 n 分别是网格的行数和列数。

```ts
function uniquePaths(m: number, n: number): number {
    const memo = Array.from({ length: m + 1 }, () => Array(n + 1).fill(0))
    memo[0][0] = 1
    for (let i = 1; i < m + 1; i++) {
        for (let j = 1; j < n + 1; j++) {
            if (i === 1 && j === 1) {
                memo[1][1] = 1
            } else {
                memo[i][j] = memo[i - 1][j] + memo[i][j - 1]
            }
        }
    }
    return memo[m][n]
};
```

### 3.空间优化
实际上就是一列一列第刷过每个格子去累加每个格子的走法，最终到达右下角求得的总和就是答案。

- 时间复杂度：$O(mn)$，其中 m 和 n 分别是网格的行数和列数。
- 空间复杂度：$O(n)$，其中 n 是网格的列数。

```ts
function uniquePaths(m: number, n: number): number {
    const memo = Array(n + 1).fill(0)
    memo[1] = 1
    for (let i = 0; i < m; i++) {
        for (let j = 1; j < n + 1; j++) {
            memo[j] += memo[j - 1]
        }
    }
    
    return memo[n]
};
```
