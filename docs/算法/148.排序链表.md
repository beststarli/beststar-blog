# 148.排序链表
给你链表的头结点`head`，请将其按 **升序** 排列并返回 **排序后的链表** 。

示例1：
![148-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/148-1.jpg)
- **输入**：head = [4,2,1,3]
- **输出**：[1,2,3,4]

示例2：
![148-2](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/148-2.jpg)
- **输入**：head = [-1,5,3,4,0]
- **输出**：[-1,0,3,4,5]

示例3：
- **输入**：head = []
- **输出**：[]

**进阶**：
- 你可以在`O(n log n)`时间复杂度和常数级空间复杂度下，对链表进行排序吗？

## 解题思路
### 1.归并排序——分治
解法参考自[灵茶山艾府](https://leetcode.cn/problems/sort-list/solutions/2993518/liang-chong-fang-fa-fen-zhi-die-dai-mo-k-caei/?envType=study-plan-v2&envId=top-100-liked)。

要对一个无序链表排序，首先想到把问题最小化对两个结点进行排序，再用递归的方式一点一点解决问题。递归思想解这道题没错，但发现不好处理每次递归调用的衔接关系。

参考先前的[21.合并两个有序链表](/docs/算法/合并两个有序链表)，如果我们把原链表分成两半，对这两半链表进行排序再有序合并就达到目的了，那么将问题最小化，对分半后的链表再次分半，直到不可再分，此时将得到的多对单结点链表向上进行有序合并，那么最终就得到的完整的有序链表了。这便是归并排序，体现着分治的思想。

排序并合并的思路我们已经确定了，那么如何对链表分半呢？参考先前的[876.链表的中间结点](/docs/算法/链表的中间结点)，我们可以用快慢指针来找到链表的中间结点，从而将链表分成两半。需要注意的是，每次分半后需要断开后一半链表与前一半的连接。用一个指针维护慢指针的前一个结点，并在返回时将其next指向为null即可。

至此，解题思路已经完全确定了。

时间复杂度：O(n log n)，其中n是链表的结点数。

空间复杂度：O(logn)。递归需要O(logn)的栈开销。

```ts
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     val: number
 *     next: ListNode | null
 *     constructor(val?: number, next?: ListNode | null) {
 *         this.val = (val===undefined ? 0 : val)
 *         this.next = (next===undefined ? null : next)
 *     }
 * }
 */

function sortList(head: ListNode | null): ListNode | null {
    if (head === null || head.next === null) {
        return head
    }

    let preHalf = head
    let postHalf = middleNode(preHalf)

    preHalf = sortList(preHalf)
    postHalf = sortList(postHalf)

    return mergeTwoLists(preHalf, postHalf)
};

// 876.链表的中间结点
function middleNode(head) {
    let pre = head
    let slow = head
    let fast = head

    while (fast && fast.next) {
        pre = slow
        slow = slow.next
        fast = fast.next.next
    }

    pre.next = null
    return slow
}

// 21.合并两个有序链表
function mergeTwoLists(list1, list2) {
    const dummy = new ListNode()
    let cur = dummy
    while (list1 && list2) {
        if (list1.val < list2.val) {
            cur.next = list1
            list1 = list1.next
        } else {
            cur.next = list2
            list2 = list2.next
        }
        cur = cur.next
    }
    cur.next = list1 ? list1 : list2
    return dummy.next
}
```
### 2.归并排序——迭代
解法参考自[灵茶山艾府](https://leetcode.cn/problems/sort-list/solutions/2993518/liang-chong-fang-fa-fen-zhi-die-dai-mo-k-caei/?envType=study-plan-v2&envId=top-100-liked)。

解法1分治是自上而下求解需要O(logn)的递归栈开销，迭代则是自下而上计算，空间复杂度为O(1)。

自底向上的意思是：
- 首先，归并长度为`1`的子链表。例如`[4,2,1,3]`，把第一个节点和第二个节点归并，第三个节点和第四个节点归并，得到`[2,4,1,3]`。
- 然后，归并长度为`2`的子链表。例如`[2,4,1,3]`，把前两个节点和后两个节点归并，得到`[1,2,3,4]`。
- 然后，归并长度为`4`的子链表。
- 依此类推，直到归并的长度大于等于链表长度为止，此时链表已经是有序的了。

具体算法：
- 遍历链表，获取链表长度$length$。
- 初始化步长$step=1$。
- 循环直到$step≥length$。
- 每轮循环，从链表头节点开始。
- 分割出两段长为$step$的链表，合并，把合并后的链表插到新链表的末尾。重复该步骤，直到链表遍历完毕。
- 把$step$扩大一倍。回到第$4$步。

时间复杂度：O(nlogn)，其中 n 是链表长度。

```ts
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     val: number
 *     next: ListNode | null
 *     constructor(val?: number, next?: ListNode | null) {
 *         this.val = (val===undefined ? 0 : val)
 *         this.next = (next===undefined ? null : next)
 *     }
 * }
 */

function sortList(head: ListNode | null): ListNode | null {
    const length = getListLength(head)
    const dummy = new ListNode(0, head)

    for (let step = 1; step < length; step *= 2) {
        let newListTail = dummy
        let cur = dummy.next
        while (cur) {
            const head1 = cur
            const head2 = splitList(head1, step)
            cur = splitList(head2, step)

            const [head, tail] = mergeTwoLists(head1, head2)
            newListTail.next = head
            newListTail = tail
        }
    }
    return dummy.next
};

// 获取链表长度
function getListLength(head) {
    let length = 0
    while (head) {
        length++
        head = head.next
    }
    return length
}

// 分割链表
function splitList(head, size) {
    let cur = head
    for (let i = 0; i < size - 1 && cur; i++) {
        cur = cur.next
    }

    if (cur === null || cur.next === null) {
        return null
    }

    const nextHead = cur.next
    cur.next = null
    return nextHead
}

// 21.合并两个有序链表
function mergeTwoLists(list1, list2) {
    const dummy = new ListNode()
    let cur = dummy
    while (list1 && list2) {
        if (list1.val < list2.val) {
            cur.next = list1
            list1 = list1.next
        } else {
            cur.next = list2
            list2 = list2.next
        }
        cur = cur.next
    }
    cur.next = list1 ? list1 : list2
    while (cur.next) {
        cur = cur.next
    }
    return [dummy.next, cur]
}
```

## 总结
### 归并排序
详情见[Hello算法 11.6归并排序](https://www.hello-algo.com/chapter_sorting/merge_sort/)

归并排序（merge sort）是一种基于分治策略的排序算法，包含“划分”和“合并”两个阶段。
- **划分阶段**：通过递归不断地将数组从中点处分开，将长数组的排序问题转换为短数组的排序问题。
- **合并阶段**：当子数组长度为 1 时终止划分，开始合并，持续地将左右两个较短的有序数组合并为一个较长的有序数组，直至结束。
![148-3](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/148-3.png)

对于链表，归并排序相较于其他排序算法具有显著优势，可以将链表排序任务的空间复杂度优化至$O(1)$。
- 划分阶段：可以使用“迭代”替代“递归”来实现链表划分工作，从而省去递归使用的栈帧空间。
- 合并阶段：在链表中，节点增删操作仅需改变引用（指针）即可实现，因此合并阶段（将两个短有序链表合并为一个长有序链表）无须创建额外链表。
