# 189.轮转数组
给定一个整数数组`nums`，将数组中的元素向右轮转`k`个位置，其中`k`是非负数。

示例1：
- **输入**：nums = [1,2,3,4,5,6,7], k = 3
- **输出**：[5,6,7,1,2,3,4]
- **解释**：
```txt
向右轮转 1 步: [7,1,2,3,4,5,6]
向右轮转 2 步: [6,7,1,2,3,4,5]
向右轮转 3 步: [5,6,7,1,2,3,4]
```

示例2：
- **输入**：nums = [-1,-100,3,99], k = 2
- **输出**：[3,99,-1,-100]
- **解释**：
```txt
向右轮转 1 步: [99,-1,-100,3]
向右轮转 2 步: [3,99,-1,-100]
```

**进阶**：
- 尽可能想出更多的解决方案，至少有 三种 不同的方法可以解决这个问题。
- 你可以使用空间复杂度为`O(1)`的 **原地** 算法解决这个问题吗？

## 解题思路
### 1.尾部剔除头部添加
既然题意是每次循环将数组末尾的元素移动至数组头部，那很自然的就可以想到先对数组使用`pop()`剔除并得到元素再使用`unshift()`将元素添加至数组头部就可以了，每次循环都进行上述操作直到循环k次即可。

时间复杂度：O(k*n)，k为轮转的次数。

```ts
function rotate(nums: number[], k: number): void {
    let rotateVector = 0
    while (k > 0) {
        rotateVector = nums.pop()
        nums.unshift(rotateVector)

        k--
    }
};
```

### 2.三次反转
定义一个自定义数组反转方法，这个方法是用双指针对整个数组内的元素进行反转排列。这样用这个反转方法对原数组操作三次：
1. 反转整个数组
2. 反转前k个元素
3. 反转后k索引开始剩余的元素

时间复杂度：O(n)

空间复杂度：O(1)

```ts
function rotate(nums: number[], k: number): void {
    k = k % nums.length;           // 关键优化：k 可能远大于 n
    if (k === 0) return;

    // 1. 整体反转
    reverse(nums, 0, nums.length - 1);
    // 2. 反转前 k 个
    reverse(nums, 0, k - 1);
    // 3. 反转后面部分
    reverse(nums, k, nums.length - 1);
}

function reverse(arr: number[], start: number, end: number): void {
    while (start < end) {
        [arr[start], arr[end]] = [arr[end], arr[start]];
        start++;
        end--;
    }
}
```

### 3.裁剪再拼接
可以注意到最终输出的结果就像是原数组从中间截断后，将后半段的尾部拼接在前半段的头部。那自然也想到了使用`splice()`方法对数组做裁剪操作，再用`concat()`方法将裁剪后的数组拼接在一起即可。不过有一步将拼接数组写回原数组的操作需要注意。

时间复杂度：O(n)

```ts
function rotate(nums: number[], k: number): void {
    const n = nums.length;
    if (n <= 1) return;
   
    k = k % n;
    if (k === 0) return;
   
    // 直接用 slice 切两段再拼接
    const rotated = nums.slice(-k).concat(nums.slice(0, n - k));
   
    // 写回原数组
    for (let i = 0; i < n; i++) {
        nums[i] = rotated[i];
    }
}
```
