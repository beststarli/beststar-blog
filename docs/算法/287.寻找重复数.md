# 287.寻找重复数
给定一个包含`n + 1`个整数的数组 `nums` ，其数字都在 `[1, n]` 范围内（包括 `1` 和 `n`），可知至少存在一个重复的整数。

假设 `nums` 只有 **一个重复的整数** ，返回 **这个重复的数** 。

你设计的解决方案必须 **不修改** 数组 `nums` 且只用常量级 `O(1)` 的额外空间。

示例1：
- **输入**：nums = [1,3,4,2,2]
- **输出**：2

示例2：
- **输入**：nums = [3,1,3,4,2]
- **输出**：3

示例3：
- **输入**：nums = [3,3,3,3,3]
- **输出**：3

**进阶**：
- 如何证明 `nums` 中至少存在一个重复的数字?
- 你可以设计一个线性级时间复杂度 `O(n)` 的解决方案吗？

## 解题思路
### 1.基环树
有一个`n + 1`个节点的图，编号从`0`到`n`，对`i = 0, 1, 2, ..., n`，连一条从`i`到`nums[i]`的有向边，可以得到一个有向图。

![287-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/287-1.png)

每个节点的入度，就是这个节点在`nums`中的出现次数，重复元素的入度大于1。在每个节点的出度都是1的情况下，`n + 1`个点`n+1`条边的有向图，又叫内向基环森林，每个连通块都恰好有一个环。由于`nums[i] >= 1`，所以节点`0`的入度为`0`，不在环上。从节点`0`出发，进入基环，基环的入环口就是入度大于`1`的节点。

:::warning **从非0节点出发**
如果从非`0`节点出发，可能无法找到答案。如下图，如果从节点`3`出发，无法找到入度大于`1`的节点。
![287-2](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/287-2.png)
:::

时间复杂度：$O(n)$，其中 n 是数组 nums 的长度。

```ts
function findDuplicate(nums: number[]): number {
    // 0 一定不在环上，适合作为起点
    let slow = 0
    let fast = 0
    while (true) {
        slow = nums[slow]   // 等价于 slow = slow.next
        fast = nums[nums[fast]] // 等价于 fast = fast.next.next

        // 快慢指针移动到同一个节点
        if (slow === fast) {
            break
        }
    }

    // 再用一个指针，从起点出发
    let head = 0
    while (slow !== head) {
        slow = nums[slow]
        head = nums[head]
    }
    return slow // 入环口即重复元素
};
```


## 总结
本题的逻辑类似[142.环形链表II](/docs/算法/环形链表Ⅱ)，可以使用快慢指针来找到入环口。两题的联系如下表，其中i和nums[i]都是节点：
| 142. 环形链表 II          | 287. 寻找重复数     |
|---------------------------|---------------------|
| **链表节点** `node`       | `i`                 |
| **下一个节点** `node.next`| `nums[i]`           |
| **头节点** `head`         | `0`                 |
| **入环口**                | **重复元素**        |

在链表中，移动方式是从节点node移动到下一个节点node.next。在数组中，移动方式是从节点i移动到节点nums[i]。由于数组长度是n + 1，而nums[i] < n + 1，所以下标不会越界。
