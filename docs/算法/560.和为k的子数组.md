# 560.和为k的子数组
给你一个整数数组`nums`和一个整数`k`，请你统计并返回*该数组中和为`k`的子数组的个数*。

子数组是数组中元素的连续非空序列。

示例1：
- **输入**：nums = [1,1,1], k = 2
- **输出**：2

示例2：
- **输入**：nums = [1,2,3], k = 3
- **输出**：2

## 解题思路
### 1.两层循环遍历计算目标值
因为题干说明了子数组是一个连续非空序列，那么暴力解法很简单就可以想到，用两层循环推进计算求和，当为目标值时结果加一即可。需注意外层循环元素本身就是目标值的情况

时间复杂度：O(n²)

:::warning **耗时极长**
本方法涉及两层for循环，虽可以通过全部测试用例完成解题，但实际耗时达到了**2279ms**，超级慢！！！
![560](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/560.png)
:::

```ts
function subarraySum(nums: number[], k: number): number {
    let result: number = 0

    for (let i = 0; i < nums.length; i++) {
        let sum = nums[i]
        let j = i + 1

        if (sum === k) result++

        while (j < nums.length) {
            sum += nums[j]
            if (sum === k) result++
            j++
        }
    }
    return result
};
```

### 2.前缀和与哈希map——一次遍历写法一
以题目[303.区域和检索-数组不可变](/docs/算法/303.区域和检索-数组不可变)为思路，使用前缀和与哈希map进行解题。

```ts
function subarraySum(nums: number[], k: number): number {
    let result: number = 0

    const preSum = new Map<number, number>()
    preSum.set(0, 1)

    let sum = 0
    for (const num of nums) {
        sum += num

        if (preSum.has(sum - k)) {
            result += preSum.get(sum - k)!
        }
        preSum.set(sum, (preSum.get(sum) || 0) + 1)
    }
    return result
}
```

## 二刷
### 1.前缀和与哈希map——两次遍历
用一个数组第一次遍历来记录所有前缀和，第二次遍历用一个哈希map来记录前缀和出现的次数，当前缀和减去目标值在哈希map中存在时，说明找到了和为k的子数组，结果加上该前缀和与目标值差值对应的出现的次数。

时间复杂度：O(n)。

空间复杂度：O(n)。

```ts
function subarraySum(nums: number[], k: number): number {
    const length = nums.length
    const sumArr = new Array(length + 1).fill(0)
    for (let i = 0; i < length; i++) {
        sumArr[i + 1] = sumArr[i] + nums[i]
    }

    let ans = 0
    const count = new Map()
    for (const sum of sumArr) {
        ans += (count.get(sum - k) || 0)
        count.set(sum, (count.get(sum) || 0) + 1)
    }
    return ans
};
```

### 2.前缀和与哈希map——一次遍历写法二
相较于[一次遍历写法一](/docs/算法/和为k的子数组#2前缀和与哈希map一次遍历写法一)，这次每轮遍历先更新上轮前缀和的出现频率，这样就省去了初始化`preSum.set(0, 1)`的步骤。再算本轮的前缀和，并与目标值做差来更新结果。由于最后一轮遍历计算的是数组总和，其出现的频率已经不会影响结果了，所以不需要更新哈希map了。之所以ans用+=是因为每找到一个新的符合sum-k的前缀和，就是找到了对应计数的个数的子数组。找到子数组的左边界是不同的。

- 时间复杂度：O(n)。
- 空间复杂度：O(n)。

```ts
function subarraySum(nums: number[], k: number): number {
    const count = new Map()
    let ans = 0
    let sum = 0
    
    for (const num of nums) {
        count.set(sum, (count.get(sum) || 0) + 1)
        sum += num
        ans += (count.get(sum - k) || 0)
    }
    return ans
};
```

## 总结
### 为什么不适合用滑动窗口？
滑动窗口需要满足单调性，当右端点元素进入窗口时，窗口元素和是不能减少的。本题 nums 包含负数，当负数进入窗口时，窗口左端点该往哪个方向移动？无法确定。如果没有负数的话，则可以用滑动窗口（恰好型滑动窗口）。题目[930.和相同的二元子数组](/docs/算法/和相同的二元子数组)就是一个适合用滑动窗口解题的例子。
