# 169.多数元素
给定一个大小为`n`的数组`nums`，返回其中的多数元素。多数元素是指在数组中出现次数 **大于**`⌊ n/2 ⌋`的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

示例1：
- **输入**：nums = [3,2,3]
- **输出**：3

示例2：
- **输入**：nums = [2,2,1,1,1,2,2]
- **输出**：2

**进阶**:
- 尝试设计时间复杂度为 O(n)、空间复杂度为 O(1) 的算法解决此问题。

## 解题思路

### 1.哈希map
用哈希map来统计数组中元素出现的频率，最后遍历哈希map找到出现次数大于`⌊ n/2 ⌋`的元素。

时间复杂度：O(n)。

空间复杂度：O(n)。

```ts
function majorityElement(nums: number[]): number {
    let ans = 0
    const count = new Map()
    const minShow = nums.length / 2

    for (const num of nums) {
        if (count.get(num)) {
            count.set(num, count.get(num) + 1)
        } else {
            count.set(num, 1)
        }
    }

    for (const [key, value] of count) {
        if (value > minShow) {
            ans = key
        }
    }
    return ans
}
```

### 2.原地排序
多数元素在排序后的数组中一定位于中间位置，即索引为`⌊ n/2 ⌋`的元素。

时间复杂度：O(nlogn)。

空间复杂度：O(1)。

```ts
function majorityElement(nums: number[]): number {
    nums.sort()
    return nums[Math.floor(nums.length / 2)]
};
```

### 3.进阶：Boyer-Moore投票算法
我们取数组第一个元素为”候选者“，并且初始就有1票，遇到相同的元素就把计数即”票数“加1，不同的元素就减1，当票数为0时，我们就把下一个元素作为新的候选者，并且把票数重置为1。由于多数元素出现的次数大于`⌊ n/2 ⌋`，所以最终留下的候选者就是多数元素。

时间复杂度：O(n)。

空间复杂度：O(1)。

```ts
function majorityElement(nums: number[]): number {
    let candidate = 0
    let count = 0

    for (const num of nums) {
        if (count === 0) {
            candidate = num
            count = 1
        } else [
            count += ((num === candidate) ? 1 : -1)
        ]
    }
    return candidate
};
```
