# 字母异位词分组
给你一个字符串数组，请你将**字母异位词**组合在一起。可以按任意顺序返回结果列表。

:::secondary 字母异位词
字母异位词是通过重新排列不同单词或短语的字母而形成的单词或短语，并使用所有原字母一次。
:::

示例1：
- **输入**：strs = ["eat", "tea", "tan", "ate", "nat", "bat"]
- **输出**：[["bat"],["nat","tan"],["ate","eat","tea"]]
- **解释**：
    - 在 strs 中没有字符串可以通过重新排列来形成 "bat"。
    - 字符串`"nat"`和`"tan"`是字母异位词，因为它们可以重新排列以形成彼此。
    - 字符串`"ate"`，`"eat"`和`"tea"`是字母异位词，因为它们可以重新排列以形成彼此。

示例2：
- **输入**：strs = [""]
- **输出**：[[""]]

示例3：
- **输入**：strs = ["a"]
- **输出**：[["a"]]

## 解题思路
### 个人题解
#### 1.暴力嵌套循环
最简单粗暴的，既然是要将字符串列表中的元素根据相同字母归类为同一组，然后将各组添加到列表中作为结果返回，那么依旧可以使用两层循环来实现对每个字符串元素的枚举，不过需要注意的是在第二层循环时应记录已访问被拆解并添加进组的异位词，防止第一层循环时再次访问。

时间复杂度：O(n²)
```ts
function groupAnagrams(strs: string[]): string[][] {
    let result: string[][] = []
    const visited = new Set<number>()
    
    for (let i = 0; i < strs.length; i++) {
        if (visited.has(i)) continue

        let listA: string[] = [strs[i]]
        visited.add(i)
        const sortedKey = strs[i].split('').sort().join('')
        
        for (let j = i + 1; j < strs.length; j++) {
            if (visited.has(j)) continue
            const jSortedKey = strs[j].split('').sort().join('')
            if (jSortedKey === sortedKey) { 
                listA.push(strs[j])
                visited.add(j)
            }
        }
        result.push(listA)
    }
    
    return result
}
```

#### 2.哈希map
借两数之和的解题思路，本题要是想通过哈希法来解题，那么很明显要使用map数据结构，而分割排序再连接后的字符串最为索引键，值则是拥有相同字母的异位词列表，也就是`{key:分割排序再连接后的字符串, value:[拥有相同字母的异位词]}`,通过一层循环遍历输入的字符串列表构建map，最后将map中所有的值构建列表返回即可。

时间复杂度：O(n*klogk)（k=字符串长，`sort`是logk）
```ts
function groupAnagrams(strs: string[]): string[][] {
    const map = new Map<string, string[]>()

    for (const str of strs) {
        const strSortedKey = str.split('').sort().join('')
        if (!map.has(strSortedKey)) {
            map.set(strSortedKey, [])
        }
        map.get(strSortedKey).push(str)
    }

    return Array.from(map.values())
}
```

## 总结
本题的解题思路：
- a：用哈希表的方法来做，用map来存储字母相同的单词。
- b：将每一个单词的字母排序后用作key，保证相同字母的单词的key相同，遍历一遍数组即可将所有字母相同的单词放在同一key中。
