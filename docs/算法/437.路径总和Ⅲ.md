# 437.路径总和Ⅲ
给定一个二叉树的根节点`root`，和一个整数`targetSum`，求该二叉树里节点值之和等于`targetSum`的 **路径** 的数目。

**路径** 不需要从根节点开始，也不需要在叶子节点结束，但是路径方向必须是向下的（只能从父节点到子节点）。

示例1：
![437-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/437-1.jpg)
- **输入**：root = [10,5,-3,3,2,null,11,3,-2,null,1], targetSum = 8
- **输出**：3
- **解释**：和等于 8 的路径有 3 条，如图所示。

示例2：
- **输入**：root = [5,4,8,11,null,13,4,7,2,null,null,5,1], targetSum = 22
- **输出**：3

## 解题思路
### 1.前缀和与递归
解法参考自[灵茶山艾府](https://leetcode.cn/problems/path-sum-iii/solutions/2784856/zuo-fa-he-560-ti-shi-yi-yang-de-pythonja-fmzo/?envType=study-plan-v2&envId=top-100-liked)

这道题是求二叉树上路径节点值之和等于目标值的路径数目，如果把这颗二叉树扁平化为一个数组，那么就变成了求数组中和为目标值的子数组的个数，这道题就变成了[560.和为k的子数组](/docs/算法/560.和为k的子数组)了。那么同样的思路，我们利用前缀和来求解这道题。要解决的问题是：DFS 遍历这棵树，遍历到节点`node`时，假设`node`是路径的终点，那么有多少个起点，满足起点到终点`node`的路径总和恰好等于`targetSum`？

过程是：一边遍历二叉树，一边用哈希表`count`统计前缀和（从根节点开始的路径和）的出现次数。设从根到终点`node`的路径和为`sum`，那么起点的个数就是`count[sum - targetSum]`，加入答案。对比 560 题，我们在枚举子数组的右端点（终点），统计有多少个左端点（起点），做法完全一致。

时间复杂度：O(n)，其中n是树的结点数。

```ts
/**
 * Definition for a binary tree node.
 * class TreeNode {
 *     val: number
 *     left: TreeNode | null
 *     right: TreeNode | null
 *     constructor(val?: number, left?: TreeNode | null, right?: TreeNode | null) {
 *         this.val = (val===undefined ? 0 : val)
 *         this.left = (left===undefined ? null : left)
 *         this.right = (right===undefined ? null : right)
 *     }
 * }
 */

function pathSum(root: TreeNode | null, targetSum: number): number {
    const count = new Map()
    count.set(0, 1)

    let ans = 0

    function dfs(node, sum) {
        if (node === null) {
            return
        }

        sum += node.val
        ans += count.get(sum - targetSum) ?? 0

        count.set(sum, (count.get(sum) || 0) + 1)
        dfs(node.left, sum)
        dfs(node.right, sum)

        // 恢复现场，节点回退
        count.set(sum, count.get(sum) - 1)
    }

    dfs(root, 0)
    return ans
};
```
