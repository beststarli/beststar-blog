# 416.分割等和子集
给你一个 **只包含正整数** 的 **非空** 数组 `nums` 。请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

示例1：
- **输入**：nums = [1,5,11,5]
- **输出**：true
- **解释**：数组可以分割成 [1, 5, 5] 和 [11] 。

示例2：
- **输入**：nums = [1,2,3,5]
- **输出**：false
- **解释**：数组不能分割成两个元素和相等的子集。

## 解题思路
既然是将原数组均分为两个和相等的子数组，那么转变一下问题，就是从原数组挑选元素，每个元素仅能选择一次，选出的元素和为原数组元素和的一半。于是变成了01背包问题，讨论元素选还是不选。

### 1.递归加记忆化
注意到，对于原数组的和如果不能被2整除，意味着不可能找到满足题意的两个子集，直接返回`false`。否则，目标和为原数组元素和的一半。

定义`dfs(i, j)`表示在原数组前`i`个元素中，是否存在一个子集使得子集元素和为`j`。对于`nums[i]`，有两种选择：
- 不选`nums[i]`，那么`dfs(i, j) = dfs(i - 1, j)`。
- 选`nums[i]`，那么`dfs(i, j) = dfs(i - 1, j - nums[i])`。

当`nums[i] > j`时，无法选`nums[i]`，只能不选。否则，两种情况取或运算，得到`dfs(i, j) = dfs(i - 1, j) || dfs(i - 1, j - nums[i])`。

递归边界为：当`i < 0`时，如果`j === 0`，说明找到了一个子集使得子集元素和为`j`，返回`true`；否则返回`false`。

递归入口为`dfs(length - 1, sum / 2)`，其中`length`是原数组长度，`sum`是原数组元素和。

时间复杂度：$O(n \times sum)$，其中 n 是数组 nums 的长度，sum 是数组 nums 中元素的和。

#### 写法一
```ts
function canPartition(nums: number[]): boolean {
    let sum = 0
    for (const num of nums) {
        sum += num
    }

    if (sum % 2 !== 0) {
        return false
    }

    let res
    const length = nums.length
    const memo = Array.from({ length: length }, () => Array(sum / 2 + 1).fill(-1))

    function dfs(i, j) {
        if (i < 0) {
            return j === 0
        }
        if (memo[i][j] !== -1) {
            return memo[i][j] === 1
        }

        if (j < nums[i]) {
            res = dfs(i - 1, j)
        } else {
            res = dfs(i - 1, j - nums[i]) || dfs(i - 1, j)
        }
        memo[i][j] = res ? 1 : 0
        return res
    }
    
    return dfs(length - 1, sum / 2)
};
```

### 2.递推
一比一将递归的「归」部分翻译为递推，自底向上计算。

`memo[i][j]`的定义与`dfs(i, j)`相同，都表示能否从`nums[0]`到`nums[i]`中选出一个和恰好等于`j`的子序列。对应的状态转移方程为：
$$
memo[i][j] = memo[i-1][j-nums[i]] \lor memo[i-1][j]
$$

但是这样定义没有能表示递归边界`i = -1`的情况，所以在二维数组中多添加一行辅助行，把`f[i]`改为`f[i + 1]`，把`f[i - 1]`改为`f[i]`，修改后`f[i+1][j]`表示能否从`nums[0]`到`nums[i]`中选出一个和为`j`的子序列，`f[0]`对应递归边界：
$$
memo[i+1][j] = memo[i][j-nums[i]] \lor memo[i][j]
$$

初始值为`f[0][0] = true`，翻译自递归边界`dfs(-1, 0) = true`，其余值初始化填入`false`。答案是`f[length][sum / 2]`。

时间复杂度：$O(n \times sum)$，其中 n 是数组 nums 的长度，sum 是数组 nums 中元素的和。

空间复杂度：$O(n \times sum)$，其中 n 是数组 nums 的长度，sum 是数组 nums 中元素的和。

```ts
function canPartition(nums: number[]): boolean {
    let sum = 0
    for (const num of nums) {
        sum += num
    }

    if (sum % 2 !== 0) {
        return false
    }

    sum /= 2

    const length = nums.length
    const memo = Array.from({ length: length + 1 }, () => Array(sum + 1).fill(false))
    memo[0][0] = true

    for (let i = 0; i < length; i++) {
        const num = nums[i]
        for (let j = 0; j <= sum; j++) {
            memo[i + 1][j] = j >= num && memo[i][j - num] || memo[i][j]
        }
    }
    
    return memo[length][sum]
};
```

### 3.空间优化
在计算`memo[i + 1][j]`时，`memo[i][j - num]`和`memo[i][j]`都只依赖于上一行的数据，所以可以用一个一维数组来维护当前行的数据，空间复杂度降为$O(sum)$。状态转移方程为：
$$
memo[j] = memo[j] \lor memo[j - nums[i]]
$$

设前`i`个数的和为`s′`，由于子序列的元素和不可能比`s′`还大，`j`可以从`min(s′,s/2)`开始倒着枚举。比如`nums`前两个数的和等于`5`，那么我们无法在前两个数中，选出一个元素和大于`5`的子序列，所以对于`j > 5`的`memo`值，一定是`false`，无需计算。

此外，可以在循环中提前判断`memo[s/2]`是否为`true`，是就直接返回`true`。

时间复杂度：$O(n \times sum)$，其中 n 是数组 nums 的长度，sum 是数组 nums 中元素的和。

空间复杂度：$O(sum)$，其中 sum 是数组 nums 中元素的和。

```ts
function canPartition(nums: number[]): boolean {
    let sum = 0
    for (const num of nums) {
        sum += num
    }

    if (sum % 2 !== 0) {
        return false
    }

    sum /= 2

    const length = nums.length
    const memo = Array(sum + 1).fill(false)
    memo[0] = true

    let sum2 = 0
    for (const num of nums) {
        sum2 = Math.min(sum2 + num, sum)
        for (let j = sum2; j >= num; j--) {
            memo[j] = memo[j] || memo[j - num]
        }
        if (memo[sum]) {
            return true
        }
    }
    return false
};
```

### 4.bitset
把布尔数组压缩成一个二进制数，二进制数从低到高第`i`位是`0`，表示布尔数组的第`i`个元素是`false`；从低到高第`i`位是`1`，表示布尔数组的第`i`个元素是`true`。

转移方程等价于，把`memo`中的每个比特位增加`x=nums[i]`，即左移`x`位，然后跟原来`memo`计算`OR`。前者对应选`x`，后者对应不选`x`。

判断`memo[s]`是否为`true`，等价于判断`memo`的第`s`位是否为`1`，即`(memo >> s & 1) == 1`。

时间复杂度：$O(n \times sum / w)$，其中 n 是数组 nums 的长度，sum 是数组 nums 中元素的和，w为32或64。

空间复杂度：$O(sum / w)$，其中 sum 是数组 nums 中元素的和，w为32或64。

```ts
function canPartition(nums: number[]): boolean {
    let sum = 0
    for (const num of nums) {
        sum += num
    }

    if (sum % 2 !== 0) {
        return false
    }

    sum /= 2
    let memo = 1n
    for (const num of nums) {
        memo |= memo << BigInt(num)
    }
    return (memo >> BigInt(sum) & 1n) === 1n
};
```
