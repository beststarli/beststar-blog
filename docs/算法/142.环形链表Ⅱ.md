# 142.环形链表Ⅱ
给定一个链表的头节点`head`，返回链表开始入环的第一个节点。*如果链表无环，则返回`null`*。

如果链表中有某个节点，可以通过连续跟踪`next`指针再次到达，则链表中存在环。为了表示给定链表中的环，评测系统内部使用整数`pos`来表示链表尾连接到链表中的位置（**索引从 0 开始**）。**注意：`pos`不作为参数进行传递。仅仅是为了标识链表的实际情况**。

**不允许修改** 链表。

示例1：
![142-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/142-1.png)
- **输入**：head = [3,2,0,-4], pos = 1
- **输出**：返回索引为 1 的链表节点
- **解释**：链表中有一个环，其尾部连接到第二个节点。

示例2：
![142-2](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/142-2.png)
- **输入**：head = [1,2], pos = 0
- **输出**：返回索引为 0 的链表节点
- **解释**：链表中有一个环，其尾部连接到第一个节点。

示例3：
![142-3](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/142-3.png)
- **输入**：head = [1], pos = -1
- **输出**：null
- **解释**：链表中没有环。

## 解题思路
### 1.快慢指针追及
解法参考自[灵茶山艾府](https://leetcode.cn/problems/linked-list-cycle-ii/solutions/1999271/mei-xiang-ming-bai-yi-ge-shi-pin-jiang-t-nvsq/?envType=study-plan-v2&envId=top-100-liked)。

![142-4](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/142-4.png)

注意：
1. `kc−a`是从入环口开始的步数。因为`(kc−a)+a=kc`，所以从`kc−a`开始，再走`a`步，就可以走满`k`圈。
2. 慢指针从相遇点开始，移动`a`步后恰好走到入环口，但在这个过程中，可能会多次经过入环口。

时间复杂度：O(n)，其中 n 是链表的节点数。

空间复杂度：O(1)，我们只使用了两个指针来遍历链表。

```ts
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     val: number
 *     next: ListNode | null
 *     constructor(val?: number, next?: ListNode | null) {
 *         this.val = (val===undefined ? 0 : val)
 *         this.next = (next===undefined ? null : next)
 *     }
 * }
 */

function detectCycle(head: ListNode | null): ListNode | null {
    if (head === null || head.next === null) {
        return null
    }

    let slowNode = head
    let fastNode = head

    while (fastNode && fastNode.next) {
        slowNode = slowNode.next
        fastNode = fastNode.next.next

        if (fastNode === slowNode) {
            while (slowNode !== head) {
                slowNode = slowNode.next
                head = head.next
            }
            return slowNode
        }
    }
    return null
};
```
