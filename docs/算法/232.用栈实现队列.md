# [232.用栈实现队列](https://leetcode.cn/problems/implement-queue-using-stacks/description/)
请你仅使用两个栈实现先入先出队列。队列应当支持一般队列支持的所有操作（`push`、`pop`、`peek`、`empty`）：

实现 `MyQueue` 类：

- `void push(int x)` 将元素 x 推到队列的末尾
- `int pop()` 从队列的开头移除并返回元素
- `int peek()` 返回队列开头的元素
- `boolean empty()` 如果队列为空，返回 `true` ；否则，返回 `false`

**说明**：
- 你 **只能** 使用标准的栈操作 —— 也就是只有 `push to top`, `peek/pop from top`, `size`, 和 `is empty` 操作是合法的。
- 你所使用的语言也许不支持栈。你可以使用 list 或者 deque（双端队列）来模拟一个栈，只要是标准的栈操作即可。

示例1：
- **输入**：
```txt
["MyQueue", "push", "push", "peek", "pop", "empty"]
[[], [1], [2], [], [], []]
```
- **输出**：[null, null, null, 1, 1, false]
- **解释**：
```txt
MyQueue myQueue = new MyQueue();
myQueue.push(1); // queue is: [1]
myQueue.push(2); // queue is: [1, 2] (leftmost is front of the queue)
myQueue.peek(); // return 1
myQueue.pop(); // return 1, queue is [2]
myQueue.empty(); // return false
```

**进阶**：
- 你能否实现每个操作均摊时间复杂度为 `O(1)` 的队列？换句话说，执行 `n` 个操作的总时间复杂度为 `O(n)` ，即使其中一个操作可能花费较长时间。

## 解题思路
### 1.双栈实现队列
用一个栈 `inStack` 来存储新加入的元素，另一个栈 `outStack` 用来存储出队的元素。当需要出队时，如果 `outStack` 为空，就将 `inStack` 中的所有元素依次弹出并压入 `outStack`，这样就实现了队列的先进先出特性。当使用`peek`时，由于不需要弹出元素，所以在使用内部`pop`后还要把元素压回去。

- 时间复杂度：
  - `push` 操作的时间复杂度为 O(1)。
  - `pop` 和 `peek` 操作的时间复杂度为 O(n)， 但由于每个元素最多只会被移动两次（一次从 `inStack` 到 `outStack`，一次从 `outStack` 弹出），所以均摊时间复杂度为 O(1)。
- 空间复杂度：O(n)，其中 n 是队列中元素的数量。我们需要两个栈来存储这些元素。

```ts
class MyQueue {
    private inStack
    private outStack

    constructor() {
        this.inStack = []
        this.outStack = []
    }

    push(x: number): void {
        this.inStack.push(x)
    }

    pop(): number {
        if (this.outStack.length === 0) {
            while (this.inStack.length !== 0) {
                this.outStack.push(this.inStack.pop())
            }
        }
        const val = this.outStack.pop()
        return val
    }

    peek(): number {
        const val = this.pop()
        this.outStack.push(val)
        return val
    }

    empty(): boolean {
        return this.inStack.length === 0 && this.outStack.length === 0
    }
}
```
