# 84.柱状图中最大的矩形
给定`n`个非负整数，用来表示柱状图中各个柱子的高度。每个柱子彼此相邻，且宽度为`1`。

求在该柱状图中，能够勾勒出来的矩形的最大面积。

示例1：
![84-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/84-1.jpg)
- **输入**：heights = [2,1,5,6,2,3]
- **输出**：10
- **解释**：最大的矩形为图中红色区域，面积为 10

示例2：
![84-2](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/84-2.jpg)
- **输入**：heights = [2,4]
- **输出**：4

## 解题思路
### 1.单调栈——三次遍历
目标矩形的最大高度一定是数组中的某个元素的值。对数组的每个高度进行枚举，去寻找最大的宽度。
- 在`i`左侧的小于`h`的最近元素的下标`left`，如果不存在则为`−1`。求出了`left`，那么`left+1`就是矩形最左边那根柱子。如果`left = −1`，那么加一后是`0`，就是整个`heights`最左边的柱子。
- 在`i`右侧的小于`h`的最近元素的下标`right`，如果不存在则为`n`。求出了`right`，那么`right−1`就是矩形最右边那根柱子。如果`right = n`，那么减一后是`n−1`，就是整个`heights`最右边的柱子。

问题就变成了如何快速计算出每个元素的`left`和`right`。

用`left`和`right`两个数组分别记录枚举的每个高度的左右边界的下标。对于每个元素，`left`和`right`的值分别初始化为`-1`和`n`，表示不存在这样的元素。

用一个栈来维护查找到的`left`与`right`高度的下标，当向左或向右继续遍历时，如果当前高度仍大于枚举高度，则将栈顶元素推出因为后续用不到，直到找到一个高度小于枚举高度的元素或者栈空为止。此时如果栈不空，栈顶元素就是`left`或`right`，如果栈空，则说明不存在这样的元素。最终矩形的宽度就是`right−left−1`。

时间复杂度： O(n)

```ts
function largestRectangleArea(heights: number[]): number {
    const length = heights.length
    const left = Array(length).fill(-1)
    const stack = []

    for (let i = 0; i < length; i++) {
        const height = heights[i]
        while (stack.length && heights[stack[stack.length - 1]] >= height) {
            stack.pop()
        }
        if (stack.length) {
            left[i] = stack[stack.length - 1]
        }
        stack.push(i)
    }

    const right = Array(length).fill(length)
    stack.length = 0;
    for (let i = length - 1; i >= 0; i--) {
        const height = heights[i];
        while (stack.length && heights[stack[stack.length - 1]] >= height) {
            stack.pop();
        }
        if (stack.length) {
            right[i] = stack[stack.length - 1];
        }
        stack.push(i);
    }

    let ans = 0
    for (let i = 0; i < length; i++) {
        ans = Math.max(ans, heights[i] * (right[i] - left[i] - 1))
    }
    return ans
};
```

### 2.优化——两次遍历
记`right[i]`为枚举高度右侧 **小于等于** 枚举高度的最近元素的下标，如果不存在则为`n`。

修改`right[i]`的定义后，我们可以把`left`和`right`合在一起计算，从而减少一次遍历：
- 在计算`left`的过程中，如果栈顶元素`≥ heights[i]`，那么`i`就是栈顶元素的`right`。

时间复杂度: O(n)

```ts
function largestRectangleArea(heights: number[]): number {
    const length = heights.length
    const left = Array(length).fill(-1)
    const right = Array(length).fill(length)
    const stack = []

    for (let i = 0; i < length; i++) {
        const height = heights[i]
        while (stack.length && heights[stack[stack.length - 1]] >= height) {
            right[stack.pop()] = i
        }
        if (stack.length) {
            left[i] = stack[stack.length - 1]
        }
        stack.push(i)
    }

    let ans = 0
    for (let i = 0; i < length; i++) {
        ans = Math.max(ans, heights[i] * (right[i] - left[i] - 1))
    }
    return ans
};
```

### 3.再优化——一次遍历
[解法2](/docs/算法/柱状图中最大的矩形#2优化两次遍历)中说明，栈顶出栈时，当前下标就是栈顶的 `right`，可否一步到位也求出栈顶的`left`?

由于单调栈是底小顶大的，栈顶下面那个柱子的高度一定比栈顶小，所以栈顶下面的值就是`left`。为简化代码逻辑，可以在一开始把`−1`入栈，当作哨兵。当栈中只有一个数的时候，栈顶下面那个数刚好就是`−1`，对应`left[i]=−1`的情况。

此外，循环结束的时候，栈中还有数据，这些数据也要计算矩形面积。处理这种情况可以再写一个循环，但更简单的办法是，往`heights`的末尾加一个`−1`（或者任意`≤ min(heights)`的数），从而保证循环结束的时候，栈一定是空的（不包括哨兵）。

时间复杂度：O(n)。

```ts
function largestRectangleArea(heights: number[]): number {
    heights.push(-1)
    const stack = [-1]
    let ans = 0
    for (let right = 0; right < heights.length; right++) {
        const height = heights[right]
        while (stack.length > 1 && heights[stack[stack.length - 1]] >= height) {
            const i = stack.pop()
            const left = stack[stack.length - 1]
            ans = Math.max(ans, heights[i] * (right - left - 1))
        }
        stack.push(right)
    }
    return ans
};
```
