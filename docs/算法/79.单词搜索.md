# 79.单词搜索
给定一个`m x n`二维字符网格`board`和一个字符串单词`word`。如果`word`存在于网格中，返回`true`；否则，返回`false`。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母不允许被重复使用。

示例1：
![79-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/79-1.jpg)
- **输入**：board = [['A','B','C','E'],['S','F','C','S'],['A','D','E','E']], word = "ABCCED"
- **输出**：true

- **输入**：board = [['A','B','C','E'],['S','F','C','S'],['A','D','E','E']], word = "SEE"
- **输出**：true

示例3：
![79-3](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/79-3.jpg)
- **输入**：board = [['A','B','C','E'],['S','F','C','S'],['A','D','E','E']], word = "ABCB"
- **输出**：false

## 解题思路
### 1.记录坐标逐步推进——基本思路
参考[200.岛屿数量](/docs/算法/200.岛屿数量)，我们可以使用深度优先搜索（DFS）来解决这个问题。我们从网格中的每个单元格开始，尝试匹配单词的第一个字符。如果匹配成功，我们继续在相邻的单元格中搜索下一个字符。为了避免重复使用同一个单元格，我们可以在搜索过程中将当前单元格标记为`'0'`代表已访问，注意如果没有搜索成功需要将其恢复，成功则保持标记。

时间复杂度：O(m*n*4^k)，其中m和n分别是网格的行数和列数，k是单词的长度。最坏情况下，我们需要从每个单元格开始搜索，并且每个字符都可能有4个方向的选择。

```ts
function exist(board: string[][], word: string): boolean {
    const height = board.length
    const length = board[0].length

    function dfs(i, j, k) {
        // 字符不匹配
        if (board[i][j] !== word[k]) {
            return false
        }
        // 单词完全匹配
        if (k + 1 === word.length) {
            return true
        }

        // 标记已访问
        board[i][j] = '0'

        // 向周围搜索
        for (const [x, y] of [[i, j - 1], [i, j + 1], [i - 1, j], [i + 1, j]]) {
            // 成功找到下个字符
            if (x >= 0 && x < height && y >= 0 && y < length && dfs(x, y, k + 1)) {
                return true
            }
        }
        // 没有找到下个字符，恢复现场
        board[i][j] = word[k]
        return false
    }

    // 主要寻找逻辑
    for (let i = 0; i < height; i++) {
        for (let j = 0; j < length; j++) {
            if (dfs(i, j, 0)) {
                return true
            }
        }
    }

    return false
};
```

### 2.优化
对[解法1](/docs/算法/79.单词搜索#1记录坐标逐步推进基本思路)的思路进行优化。

一般地，如果`word`的某个字母的出现次数，比`board`中的这个字母的出现次数还要多，可以直接返回`false`。

如果`word=abcd`但`board`中的`a`很多，`d`很少（比如只有一个），那么从`d`开始搜索，能更快地找到答案。设`word`的第一个字母在`board`中出现了`x`次，`word`的最后一个字母在`board`中出现了`y`次。如果`y<x`，我们可以把`word`反转，相当于从`word`的最后一个字母开始搜索，这样更容易在一开始就满足`board[i][j] != word[k]`，不会往下递归，递归的总次数更少。

时间复杂度：O(m*n*4^k)，其中m和n分别是网格的行数和列数，k是单词的长度。最坏情况下，我们需要从每个单元格开始搜索，并且每个字符都可能有4个方向的选择。不过通过优化，实际的搜索次数会大大减少。

```ts
function exist(board: string[][], word: string): boolean {
    const height = board.length
    const length = board[0].length

    // 优化一：字母出现频率比较
    const count = new Map()
    for (const row of board) {
        for (const letter of row) {
            count.set(letter, (count.get(letter) ?? 0) + 1)
        }
    }

    const wordCount = new Map()
    for (const letter of word) {
        wordCount.set(letter, (wordCount.get(letter) ?? 0) + 1)
        if (wordCount.get(letter) > (count.get(letter) ?? 0)) {
            return false
        }
    }

    // 优化二：比较首尾字母出现频率
    if ((count.get(word[word.length - 1]) ?? 0) < (count.get(word[0]) ?? 0)) {
        word = word.split('').reverse().join('')
    }

    function dfs(i, j, k) {
        // 字符不匹配
        if (board[i][j] !== word[k]) {
            return false
        }
        // 单词完全匹配
        if (k + 1 === word.length) {
            return true
        }

        // 标记已访问
        board[i][j] = '0'

        // 向周围搜索
        for (const [x, y] of [[i, j - 1], [i, j + 1], [i - 1, j], [i + 1, j]]) {
            // 成功找到下个字符
            if (x >= 0 && x < height && y >= 0 && y < length && dfs(x, y, k + 1)) {
                return true
            }
        }
        // 没有找到下个字符，恢复现场
        board[i][j] = word[k]
        return false
    }

    // 主要寻找逻辑
    for (let i = 0; i < height; i++) {
        for (let j = 0; j < length; j++) {
            if (dfs(i, j, 0)) {
                return true
            }
        }
    }

    return false
};
```
