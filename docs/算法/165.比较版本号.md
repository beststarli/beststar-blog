# 165.比较版本号
给你两个 **版本号字符串** `version1` 和 `version2` ，请你比较它们。版本号由被点 `'.'` 分开的修订号组成。**修订号的值** 是它 **转换为整数** 并忽略前导零。

比较版本号时，请按 **从左到右的顺序** 依次比较它们的修订号。如果其中一个版本字符串的修订号较少，则将缺失的修订号视为 `0`。

返回规则如下：
- 如果 `version1 < version2` 返回 `-1`，
- 如果 `version1 > version2` 返回 `1`，
- 除此之外返回 `0`。

示例1：
- **输入**：version1 = "1.2", version2 = "1.10"
- **输出**：-1
- **解释**：version1 的第二个修订号为 "2"，version2 的第二个修订号为 "10"：2 < 10，所以 version1 < version2。

示例2：
- **输入**：version1 = "1.01", version2 = "1.001"
- **输出**：0
- **解释**：忽略前导零，"01" 和 "001" 都代表相同的整数 "1"。

示例3：
- **输入**：version1 = "1.0", version2 = "1.0.0.0"
- **输出**：0
- **解释**：version1 有更少的修订号，每个缺失的修订号按 "0" 处理。

## 解题思路
### 1.一次遍历
首先将两个版本号字符串分别通过 `split('.')` 方法分割成数组，然后只用一个循环同时遍历两个数组，比较对应位置的修订号，注意要将字符串转换为整数并忽略前导零。循环结束后，如果没有返回结果，则说明两个版本号相等，返回 `0`。

跳出循环的条件是：`i < length1 || i < length2`，即确保两个版本号都完全遍历完，这样对于先跑完的的版本号对应位置会被`parseInt(undefined)`返回`NaN`，在比较时会被当做`0`处理。

- 时间复杂度：O(max(m, n))，其中 m 和 n 分别是 version1 和 version2 中修订号的数量。
- 空间复杂度：O(1)，只使用了常数级别的额外空间来存储指针和临时变量。

```ts
function compareVersion(version1: string, version2: string): number {
    const vArr1 = version1.split('.')
    const vArr2 = version2.split('.')
    const length1 = vArr1.length
    const length2 = vArr2.length

    for (let i = 0; i < length1 || i < length2; i++) {
        let num1 = 0
        let num2 = 0
        if (i < length1) {
            num1 = parseInt(vArr1[i])
        }
        if (i < length2) {
            num2 = parseInt(vArr2[i])
        }
        // parseInt(undefined) = NaN
        // NaN在进行比较时会被当做0处理
        if (num1 > num2) {
            return 1
        }
        if (num1 < num2) {
            return -1
        } 
    }
    return 0
};
```

## 总结
关键要想到`parseInt()`而不是用`Number()`，因为`parseInt()`会忽略前导零，而`Number()`不会。另外`parseInt(undefined)`返回`NaN`，在比较时`NaN`会被当作0来处理，即将缺失的修订号视为0。
