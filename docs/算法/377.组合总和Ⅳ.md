# 377.组合总和Ⅳ
给你一个由 **不同** 整数组成的数组`nums`，和一个目标整数`target`。请你从`nums`中找出并返回总和为`target`的元素排列的个数。

题目数据保证答案符合 32 位整数范围。

示例1：
- **输入**：nums = [1,2,3], target = 4
- **输出**：7
- **解释**：
```txt
所有可能的组合为：
(1, 1, 1, 1)
(1, 1, 2)
(1, 2, 1)
(1, 3)
(2, 1, 1)
(2, 2)
(3, 1)
请注意，顺序不同的序列被视作不同的组合。
```

示例2：
- **输入**：nums = [9], target = 3
- **输出**：0

**进阶**：
- 如果给定的数组中含有负数会发生什么？问题会产生何种变化？如果允许负数出现，需要向题目中添加哪些限制条件？

## 解题思路
本质上是[70.爬楼梯](/docs/算法/爬楼梯)的变体。

### 1.递归加记忆化
定义`dfs(i)`为寻找目标数`i`的`nums`元素排列个数，递归入口是`dfs(target)`也就是要求的答案。我们在求为到达`target`一共有多少种组合，事实上也是在求差最后`i`步到达`target`的组合数总和。因为要求刚好到达，所以如果遍历的`num`大于`i`了就不能再继续往下走了；如果`num`小于等于`i`了，就继续往下走，直到`i`为`0`了，说明找到了一种组合了。用一个记忆数组来存储已经计算过的子问题的结果，避免重复计算。

```ts
function combinationSum4(nums: number[], target: number): number {
    const memo = Array(target + 1).fill(-1)

    function dfs(i) {
        if (i === 0) {
            return 1
        }
        if (memo[i] !== -1) {
            return memo[i]
        }

        let res = 0
        for (const num of nums) { // 枚举所有可以爬的台阶数
            // 当前爬上台阶不或刚好能到顶
            if (num <= i) {
                res += dfs(i - num);
            }
        }
        return memo[i] = res; // 记忆化
    }

    return dfs(target)
};
```

### 2.递推
对[解法1](/docs/算法/组合总和Ⅳ#1递归加记忆化)的「归」过程进行翻译，改为自下向上计算。定义一个数组`memo`，其中`memo[i]`表示寻找目标数i的nums元素排列个数。初始化`memo[0] = 1`，因为当目标数为0时，只有一种组合方式，就是不选任何元素。对于每个目标数i，从1到target，枚举所有可以爬的台阶数，如果当前爬上台阶不能或刚好能到顶了，就将`memo[i]`加上`memo[i - num]`。

时间复杂度：O(n*target)，其中n是数组nums的长度，target是目标整数。每个子问题最多被计算一次，每次计算需要枚举nums数组中的所有元素，时间复杂度为O(n)。

```ts
function combinationSum4(nums: number[], target: number): number {
    const memo = Array(target + 1).fill(0)
    memo[0] = 1

    for (let i = 1; i <= target; i++) {
        for (const num of nums) {
            if (num <= i) {
                memo[i] += memo[i - num]
            }
        }
    }
    return memo[target]
};
```

这个解法与[518.零钱兑换II](/docs/算法/零钱兑换II)的解法非常类似，区别在于[518.零钱兑换II](/docs/算法/零钱兑换II)是求组合数，所以外层循环是枚举可选的硬币，内层循环是枚举目标金额；而本题是求排列数，所以外层循环是枚举目标金额，内层循环是枚举可选的元素。

## 总结
本题计算的是排列，完全背包计算的是组合。比如 1,2 和 2,1 这两个排列，在本题是有区别的，是两种方案；但在完全背包中这两个排列没有区别，只算一种方案。从代码上看，计算排列（本题）需要外层循环枚举体积，内层循环枚举物品；计算组合（完全背包）需要外层循环枚举物品，内层循环枚举体积。
