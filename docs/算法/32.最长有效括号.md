# 32.最长有效括号
给你一个只包含 `'('` 和 `')'` 的字符串，找出最长有效（格式正确且连续）括号 **子串** 的长度。

左右括号匹配，即每个左括号都有对应的右括号将其闭合的字符串是格式正确的，比如 `"(()())"`。

示例1：
- **输入**：s = "(()"
- **输出**：2
- **解释**：最长有效括号子串是 "()"

示例2：
- **输入**：s = ")()())"
- **输出**：4
- **解释**：最长有效括号子串是 "()()"

示例3：
- **输入**：s = ""
- **输出**：0

## 解题思路
### 1.栈与标记
最大长度更新做法与[485.最大连续1的个数](/docs/算法/485.最大连续1的个数)类似，

用一个栈来维护左括号的下标，再用一个数组来维护当前字符的是否合理，在遍历时如果遇到匹配的右括号，将栈顶左括号弹出，与弹出的左括号下标同时在合理性数组中更新为`true`。最后遍历合理性数组，统计连续的`true`的最大长度。    

```ts
function longestValidParentheses(s: string): number {
    const stack = []
    const length = s.length
    const isValid = Array(length).fill(false)

    // 标记可以配对的括号
    for (let i = 0; i < length; i++) {
        if (s[i] === '(') {
            stack.push(i)   // 存入左括号下标
        } else if (stack.length > 0) {
            // 右括号与栈顶左括号配对
            isValid[i] = isValid[stack.pop()] = true
        }
    }

    let ans = 0
    let count = 0
    for (const valid of isValid) {
        if (valid) {
            count++
            ans = Math.max(ans, count)
        } else {
            count = 0
        }
    }
    return ans
};
```

### 2.优化——一次遍历
既然栈中保存的是未配对的下标，那么从栈顶加一的位置到 i，就是已配对的连续括号，长度为 i 减去栈顶，更新答案的最大值。

考虑特殊情况：
- `s=)()`，其中 `s[0]` 是永远无法配对的右括号。把这种括号的下标入栈，从而保证「栈顶加一」是有效括号的左端点。
- 如果栈为空呢？此时有效括号的左端点是 `0`。我们可以在一开始，往栈中添加一个 `−1`，这样可以兼容「栈顶加一」是有效括号的左端点，从而简化代码逻辑。

时间复杂度：$O(n)$，其中 n 是字符串 s 的长度。

空间复杂度：$O(n)$，其中 n 是字符串 s 的长度。

```ts
function longestValidParentheses(s: string): number {
    const stack = [-1]  // 未配对括号的下标，其中栈底元素表示永远无法配对的括号下标
    let ans = 0

    for (let i = 0; i < s.length; i++) {
        if (s[i] === '(') {
            stack.push(i)   // 保存左括号的下标
        } else if (stack.length > 1) {
            stack.pop()     // 右括号与栈顶的左括号配对
            ans = Math.max(ans, i - stack[stack.length - 1])    // 从 st[st.length-1]+1 到 i 都已配对，长为 i - st[st.length-1]
        } else {
            // s[i] 是永远无法配对的右括号
            stack[0] = i    // 替换栈底
        }
    }

    return ans
};
```

### 3.空间优化
对于有效括号字符串，左括号和右括号的个数是相等的。能不能只记录左右括号的个数，并在左括号个数等于右括号个数时，更新答案？

设 left 为左括号的个数，`right` 为右括号的个数，分类讨论：
- 如果 `left=right`，有效括号长度为 `right⋅2`。
- 如果 `left<right`，右括号太多了，重置计数器 `left = right = 0`。
- 如果 `left>right`，我们无法只根据 `left` 和 `right` 就确定有效括号的长度。比如 `left=3`，`right=2`，那么 `s` 是 `((())` 还是 `()(()`？有效括号长为 `4` 还是 `2`？如何解决 `left>right` 这种左括号很多的情况呢？

把 `s` 旋转 `180°`，多出来的左括号就移到了后面，比如 `((())` 变成 `(()))`。这样我们就能甩掉红色括号的干扰，在遍历过程中遇到 `left = right = 2` 的情况，成功算出答案 `4`。所以再倒着遍历一遍 `s`，就不会算漏了。

时间复杂度：$O(n)$，其中 n 是字符串 s 的长度。

空间复杂度：$O(1)$，只需要常数空间存储若干变量。

```ts
function longestValidParentheses(s: string): number {
    let ans = 0
    let left = 0
    let right = 0

    for (const char of s) {
        if (char === '(') {
            left++
        } else {
            right++
        }
        // 右括号太多了，重置计数器
        if (left < right) {
            left = right = 0
        } else if (left === right) {
            ans = Math.max(ans, right * 2)
        }
    }

    left = right = 0
    for (let i = s.length - 1; i > 0; i--) {
        if (s[i] === ')') {
            left++
        } else {
            right++;
        }
        // 右括号太多了，重置计数器
        if (left < right) {
            left = right = 0;
        } else if (left === right) {
            ans = Math.max(ans, right * 2);
        }
    }

    return ans
};
```
