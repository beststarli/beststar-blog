# 234.回文链表
给你一个单链表的头节点`head`，请你判断该链表是否为回文链表。如果是，返回`true`；否则，返回`false`。
:::info **回文序列**
**回文** 序列是向前和向后读都相同的序列。
:::

示例1：
![234-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/234-1.jpg)
- **输入**：head = [1,2,2,1]
- **输出**：true

示例2：
![234-2](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/234-2.jpg)
- **输入**：head = [1,2]
- **输出**：false

**进阶**：
- 你能否用`O(n)`时间复杂度和`O(1)`空间复杂度解决此题？

## 解题思路
### 1.找中点、反转、比较
回文字符串的判断是通过首位两个指针向中间靠拢来实现的。对于链表来说，我们也可以使用类似的方法来判断是否为回文链表。具体步骤如下：
1. 使用快慢指针找到链表的中点。快指针每次移动两步，慢指针每次移动一步。当快指针到达链表末尾时，慢指针正好位于链表的中点。查找链表中点可以参考[876.链表的中间节点](/docs/算法/876.链表的中间节点.md)。
2. 反转链表的后半部分。我们可以从慢指针所在的位置开始反转链表的后半部分。反转链表思路可以参考[206.反转链表](/docs/算法/206.反转链表.md)。
3. 比较前半部分和反转后的后半部分。我们可以使用两个指针分别指向前半部分和后半部分的头节点，逐个比较它们的值。如果所有对应节点的值都相同，则链表是回文链表；否则，不是。

时间复杂度：O(n)，其中 n 是链表的节点数。我们需要遍历链表一次来找到中点，反转链表一次，以及比较链表一次。

```ts
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     val: number
 *     next: ListNode | null
 *     constructor(val?: number, next?: ListNode | null) {
 *         this.val = (val===undefined ? 0 : val)
 *         this.next = (next===undefined ? null : next)
 *     }
 * }
 */

function isPalindrome(head: ListNode | null): boolean {
    let slowNode = head
    let fastNode = head

    // 找到链表的中点
    while (fastNode !== null && fastNode.next !== null) {
        slowNode = slowNode.next!
        fastNode = fastNode.next.next
    }

    // 反转链表的后半部分
    let prev: ListNode | null = null
    let cur: ListNode | null = slowNode

    while (cur !== null) {
        const nextTemp: ListNode | null = cur.next
        cur.next = prev
        prev = cur
        cur = nextTemp
    }

    // 比较前半部分和反转后的后半部分
    let preListNode = head
    let reversePostHalfListNode = prev

    while (reversePostHalfListNode !== null) {
        if (preListNode.val !== reversePostHalfListNode.val) {
            return false
        }
        preListNode = preListNode.next!
        reversePostHalfListNode = reversePostHalfListNode.next
    }
    return true
};
```
