# 328.奇偶链表
给定单链表的头节点`head`，将所有索引为奇数的节点和索引为偶数的节点分别分组，保持它们原有的相对顺序，然后把偶数索引节点分组连接到奇数索引节点分组之后，返回重新排序的链表。

第一个节点的索引被认为是 **奇数** ， 第二个节点的索引为 **偶数** ，以此类推。

请注意，偶数组和奇数组内部的相对顺序应该与输入时保持一致。

你必须在`O(1)`的额外空间复杂度和`O(n)`的时间复杂度下解决这个问题。

示例1：
![328-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/328-1.jpg)
- **输入**：head = [1,2,3,4,5]
- **输出**：[1,3,5,2,4]

示例2：
![328-2](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/328-2.jpg)
- **输入**：head = [2,1,3,5,6,4,7]
- **输出**：[2,3,6,7,1,5,4]

## 解题思路
### 1.双指针拼接
注意到本题拼接的是索引的奇偶而不是结点的值的奇偶，那么不难发现在原链表中奇数索引结点的next.next就是下一个奇数索引结点，偶数索引结点的next.next就是下一个偶数索引结点。因此我们可以使用两个指针分别指向奇数索引结点和偶数索引结点，遍历链表将它们分别串成两条链表，最后将偶数索引链表接到奇数索引链表的末尾即可。为了方便拼接和结果返回，维护两个守卫结点即可。

时间复杂度：O(n)，其中n是链表的长度。

```ts
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     val: number
 *     next: ListNode | null
 *     constructor(val?: number, next?: ListNode | null) {
 *         this.val = (val===undefined ? 0 : val)
 *         this.next = (next===undefined ? null : next)
 *     }
 * }
 */

function oddEvenList(head: ListNode | null): ListNode | null {
    if (head === null) return null

    const newHead1 = head
    const newHead2 = head.next
    
    let p1 = newHead1
    let p2 = newHead2

    while (p2 !== null && p2.next !== null) {
        p1.next = p1.next.next
        p2.next = p2.next.next

        p1 = p1.next
        p2 = p2.next
    }
    p1.next = newHead2
    return newHead1
};
```
