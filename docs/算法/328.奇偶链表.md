# 328.奇偶链表
给定单链表的头节点`head`，将所有索引为奇数的节点和索引为偶数的节点分别分组，保持它们原有的相对顺序，然后把偶数索引节点分组连接到奇数索引节点分组之后，返回重新排序的链表。

第一个节点的索引被认为是 **奇数** ， 第二个节点的索引为 **偶数** ，以此类推。

请注意，偶数组和奇数组内部的相对顺序应该与输入时保持一致。

你必须在`O(1)`的额外空间复杂度和`O(n)`的时间复杂度下解决这个问题。

示例1：
![328-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/328-1.jpg)
- **输入**：head = [1,2,3,4,5]
- **输出**：[1,3,5,2,4]

示例2：
![328-2](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/328-2.jpg)
- **输入**：head = [2,1,3,5,6,4,7]
- **输出**：[2,3,6,7,1,5,4]

## 总结
这道题的题干意思是创建一个新链表，新链表的结点、下个结点、随机指向结点与原链表完全相同。这道题的卡点在于复制结点时，由于随机指针的存在，复制的结点的随机指针指向的结点可能还没创建，如何解决这个问题是解题的关键，要做的就是先确保各个结点都存在，也就是先把原链表的结点复制出来，再去处理各个结点的随机指针的指向关系。
