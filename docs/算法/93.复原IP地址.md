# [93.复原IP地址](https://leetcode.cn/problems/restore-ip-addresses/description/)
**有效 IP 地址** 正好由四个整数（每个整数位于 0 到 255 之间组成，且不能含有前导 0），整数之间用 `'.'` 分隔。

例如：`"0.1.2.201"` 和 `"192.168.1.1"` 是 **有效 IP** 地址，但是 `"0.011.255.245"`、`"192.168.1.312"` 和 `"192.168@1.1"` 是 **无效 IP** 地址。
给定一个只包含数字的字符串 `s` ，用以表示一个 IP 地址，返回所有可能的 **有效 IP** 地址，这些地址可以通过在 `s` 中插入 `'.'` 来形成。你 **不能** 重新排序或删除 `s` 中的任何数字。你可以按 **任何** 顺序返回答案。


示例1：
- **输入**：s = "25525511135"
- **输出**：["255.255.11.135","255.255.111.35"]

示例2：
- **输入**：s = "0000"
- **输出**：["0.0.0.0"]

示例3：
- **输入**：s = "101023"
- **输出**：["1.0.10.23","1.0.102.3","10.1.0.23","10.10.2.3","101.0.2.3"]

## 解题思路
### 1.回溯时有效性判断
对于一个IP地址来说一共分为4段，那么推入ans的path的长度需为4，且此时整个字符串s也全部遍历完毕。对于递归函数内的循环体，结束循环的条件不只是索引超出s的长度，还有索引超出当前段的长度，IP地址每段的长度最大为3。对于每段的有效性判断，首先判断当前段的整数值是否大于255，其次判断当前段是否有前导0，如果有则不合法。

- 时间复杂度：O(3^4)，因为每段最多有3种选择，且一共4段，所以总的时间复杂度为O(3^4)。
- 空间复杂度：O(1)，因为使用了常数级别的额外空间来存储结果和路径。

```ts
function restoreIpAddresses(s: string): string[] {
    const ans = []
    const path = []
    function dfs(index) {
        if (path.length === 4 && index === s.length) {
            ans.push(path.join('.'))
            return 
        }
        for (let i = index; i < s.length && i < index + 3; i++) {
            const str = s.substring(index, i + 1)
            if (parseInt(str) > 255) {
                continue
            }
            if (str.length > 1 && str[0] === '0') {
                continue
            }
            path.push(str)
            dfs(i + 1)
            path.pop()
        }
    }
    dfs(0)
    return ans
};
```
