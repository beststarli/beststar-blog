# 994.腐烂的橘子
在给定的`m x n`网格`grid`中，每个单元格可以有以下三个值之一：
- 值`0`代表空单元格；
- 值`1`代表新鲜橘子；
- 值`2`代表腐烂的橘子。
每分钟，腐烂的橘子 **周围 4 个方向上相邻** 的新鲜橘子都会腐烂。

返回 *直到单元格中没有新鲜橘子为止所必须经过的最小分钟数。如果不可能，返回 `-1`*。

示例1：
![994-1](https://blog-1385521233.cos.ap-guangzhou.myqcloud.com/docs/algorithm/994-1.png)
- **输入**：grid = [[2,1,1],[1,1,0],[0,1,1]]
- **输出**：4

示例2：
- **输入**：grid = [[2,1,1],[0,1,1],[1,0,1]]
- **输出**：-1
- **解释**：左下角的橘子（第 2 行， 第 0 列）永远不会腐烂，因为腐烂只会发生在 4 个方向上。

示例3：
- **输入**：grid = [[0,2]]
- **输出**：0
- **解释**：因为 0 分钟时已经没有新鲜橘子了，所以答案就是 0 。

## 解题思路
### 1.多源BFS
这道题的过程与[200.岛屿数量](/docs/算法/200.岛屿数量.md)十分类似，都是由网格向上下左右四个方向扩散的过程，区别在于这道题是由多个源点（腐烂的橘子）同时向外扩散，而[200.岛屿数量](/docs/算法/200.岛屿数量.md)是由一个源点（陆地）向外扩散。难点在于分钟数更新的时机，我们用一个`bad`数组来存储上一分钟数下的腐烂橘子，在下一分钟，用一个`temp`暂存上一分钟的腐烂橘子，然后将`bad`数组清空以统计新被传染的橘子。对于每一个腐烂的橘子，我们检查它上下左右四个方向，如果是新鲜橘子，则将其变为腐烂橘子，并加入`bad`数组中。每次循环结束后，分钟数加1，直到没有新鲜橘子或者没有新的腐烂橘子。

时间复杂度：O(mn)，其中m和n分别是网格的行数和列数。

```ts
function orangesRotting(grid: number[][]): number {
    const height = grid.length
    const width = grid[0].length

    let bad = []
    let fresh = 0

    for (let i = 0; i < height; i++) {
        for (let j = 0; j < width; j++) {
            if (grid[i][j] === 1) {
                fresh++
            } else if (grid[i][j] === 2) {
                bad.push([i, j])
            }
        }
    }

    let ans = 0;

    while (fresh && bad.length) {
        ans++
        const temp = bad
        bad = []
        
        for (const [x, y] of temp) {
            for (const [i, j] of [[x - 1, y], [x + 1, y], [x, y - 1], [x, y + 1]]) {
                if (i >= 0 && i < height && j >= 0 && j < width && grid[i][j] === 1) {
                    fresh--
                    grid[i][j] = 2
                    bad.push([i, j])
                }
            }
        }
    }

    return fresh ? -1 : ans
};
```
