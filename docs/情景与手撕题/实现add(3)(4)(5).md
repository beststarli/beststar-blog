事实上是实现函数的柯里化。

## 暴力版
```js
function add(a) {
  return function(b) {
    return function(c) {
      return a + b + c
    }
  }
}

console.log(add(3)(4)(5)) // 12
```

## [柯里化](/docs/情景与手撕题/柯里化currying的实现.md)
### 参数长度固定
```js
const add = function(m) {
  let temp = function(n) {
    return add(m + n)
  }
  temp.toString = function() {
    return m
  }
  return temp
}

console.log(add(3)(4)(5)) // 12
```
对于add(3)(4)(5)来说，其执行过程如下：
1. 先执行add(3)，此时m=3，返回temp函数
2. 执行temp(4)，这个函数内执行add(m+n)，n是此次传进来的数值4，m值还是上一步中的3，所以add(m+n)=add(3+4)=add(7)，此时m=7，并且返回temp函数
3. 执行temp(5)，这个函数内执行add(m+n)，n是此次传进来的数值5，m值还是上一步中的7，所以add(m+n)=add(7+5)=add(12)，此时m=12，并且返回temp函数
4. 由于后面没有传入参数，等于返回的temp函数不被执行而是打印，代码中temp函数的toString函数return m值，而m值是最后一步执行函数时的值m=12，所以返回值是12。

### 参数长度不固定
```js
function add(...args) {
  return args.reduce((a, b) => a + b)
}

function currying(fn) {
  let args = []
  return function temp(...newArgs) {
    if (newArgs.length) {
      args = [...args, ...newArgs]
      return temp
    } else {
      let val = fn.apply(this, args)
      args = [] // 保证每次调用时清空
      return val
    }
  }
}

let addCurry = currying(add)
console.log(addCurry(3)(4)(5)()) // 12
console.log(addCurry(3, 4)(5)()) // 12
console.log(addCurry(3)(4, 5)()) // 12
```
