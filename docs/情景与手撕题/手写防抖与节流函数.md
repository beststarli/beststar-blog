## 防抖
函数防抖是指在事件被触发n秒后再执行回调，如果在这n秒内事件又被触发，则重新计时。这可以使用在一些点击请求的事件上，避免因为用户的多次点击向后端发送多次请求。
```js
function debounce(func, delay) {
  let timer = null

  return function() {
    let context = this
    let args = arguments

    // 如果此时存在定时器，则取消之前的定时器重新计时
    if (timer) {
      clearTimeout(timer)
      timer = null
    }

    // 设置定时器，使时间间隔指定时间后执行回调函数
    timer = setTimeout(() => {
      func.apply(context, args)
    }, delay)
  }
}
```

## 节流
函数节流是指规定一个单位时间，在这个单位时间内，只能有一次触发事件的回调函数执行，如果在同一个单位时间内某事件被触发多次，只有一次能生效。节流可以使用在scroll函数的事件监听上，通过事件节流来降低事件调用的频率。

### 基础版（时间戳，立即执行）
```js
function throttle(func, delay) {
  let currTime = Date.now()

  return function() {
    let context = this
    let args = arguments
    let nowTime = Date.now()

    // 如果两次时间间隔超过了指定时间，则执行函数。
    if (nowTime - currTime >= delay) {
      currTime = nowTime
      return func.apply(context, args)
    }
  }
}
```

### 完整版（定时器 + cancel）
```js
function throttle(fn, delay) {
  let timer = null;
  function throttled(...args) {
    if (timer != null) return;
    timer = setTimeout(() => {
      timer = null;
      fn.apply(this, args);
    }, delay);
  }
  throttled.cancel = () => {
    if (timer != null) clearTimeout(timer);
    timer = null;
  };
  return throttled;
}
```

**原理**：timer 不为空时说明在节流窗口内，直接 return。窗口结束后执行 fn 并清空 timer，准备下一个窗口。

**与基础版的区别**：基础版用时间戳判断（立即执行）；完整版用定时器（延迟执行），且暴露 cancel 方法。|
